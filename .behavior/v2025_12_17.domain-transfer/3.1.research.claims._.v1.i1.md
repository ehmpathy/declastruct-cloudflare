# research: claims

research on facts, assumptions, questions, and opinions relevant to the wish, vision, and criteria.

## category.1: ICANN domain transfer requirements

### [FACT] 60-day transfer lock

> "Your domain must have been registered at least 60 days ago and not transferred in the last 60 days."

source: [1] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

**implication**: declastruct cannot automate transfers for recently registered or recently transferred domains. HelpfulError guidance must mention this constraint.

### [FACT] one-year registration extension on transfer

> "ICANN requires that any transfer also extends the expiration date of your domain by at least one year from your current expiration date."

source: [1] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

**implication**: users should know that transfer costs include one year of registration.

### [FACT] 5-7 day transfer timeline

> "Domain transfers typically take 5-7 days as per ICANN regulations."

source: [1] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

**implication**: HelpfulError guidance must set expectations for wait time. this is not a cloudflare limitation but an ICANN requirement.

### [FACT] authorization code required

> "You will need to obtain an authorization code (also called an EPP code) from your current registrar to initiate the transfer."

source: [1] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

**implication**: transfer guidance must include step to obtain auth code from current registrar.

## category.2: cloudflare registrar API limitations

### [FACT] domain registration API is enterprise-only

> "API for domain registration is available only for Enterprise customers."

source: [2] https://community.cloudflare.com/t/register-and-manage-domains-with-cloudflare-registrar-api/453793

**implication**: confirms vision's "guided" approach. non-enterprise users cannot automate domain registration via API.

### [FACT] no new domain registration endpoint for standard users

> "At this time there is no API endpoint to register new domains."

source: [3] https://community.cloudflare.com/t/domain-registration-using-the-api/382556

**implication**: HelpfulError purchase guidance is the correct approach for non-enterprise users.

### [FACT] registrar API limited to GET for non-enterprise

> "You are limited to GET requests unless you have an Enterprise plan."

source: [2] https://community.cloudflare.com/t/register-and-manage-domains-with-cloudflare-registrar-api/453793

**implication**: even update operations may be limited. need to verify which operations work with standard API tokens.

### [FACT] zone addition rate limits

> "You can only sign up a maximum of 25 domains every 10 minutes."

source: [4] https://developers.cloudflare.com/fundamentals/manage-domains/add-multiple-sites-automation/

**implication**: bulk zone creation has rate limits. declastruct plan/apply should handle rate limiting gracefully.

## category.3: cloudflare WHOIS intel API

### [FACT] WHOIS API returns registration state

> "The API returns information including... whether the domain was found, nameservers, registrant, and registrar details."

source: [5] https://developers.cloudflare.com/api/resources/intel/subresources/whois/methods/get/

**implication**: confirms WHOIS can detect if domain is registered elsewhere, enabling smart guidance (transfer vs purchase).

### [FACT] WHOIS API endpoint

> "https://api.cloudflare.com/client/v4/accounts/{account_id}/intel/whois?domain=cloudflare.com"

source: [5] https://developers.cloudflare.com/api/resources/intel/subresources/whois/methods/get/

**implication**: exact endpoint confirmed. requires account_id scope.

## category.4: infrastructure as code best practices

### [FACT] idempotency is a core IaC principle

> "Idempotency refers to the property of certain operations where applying the operation multiple times has the same effect as applying it once."

source: [6] https://medium.com/@tiwari.sushil/idempotency-the-secret-to-seamless-devops-and-infrastructure-bf22e63e1be5

**implication**: validates declastruct's findsert/upsert semantics. operations must be safe to retry.

### [FACT] declarative approach is preferred

> "In a declarative approach, the desired state of the infrastructure is well-defined, and the IaC tool then automatically determines and executes the necessary steps to achieve this state."

source: [7] https://squareops.com/knowledge/infrastructure-code-tools-benefits-examples/

**implication**: validates declastruct pattern. users declare desired state, system figures out operations.

### [OPIN] plan before apply is essential

> "Use plan options (terraform plan, pulumi preview) to verify before applying changes."

source: [8] https://umatechnology.org/infrastructure-as-code-examples-for-multi-zone-dns-management-with-config-as-code/

**implication**: supports declastruct plan → apply workflow. plan shows changes before execution.

### [FACT] idempotency prevents configuration drift

> "An idempotent configuration ensures that the same operation multiple times has no additional effect beyond the initial application, and this predictability extinguishes configuration drift."

source: [9] https://www.techtarget.com/searchitoperations/tip/Idempotent-configuration-management-sets-things-right-no-matter-what

**implication**: declastruct's drift detection (plan shows delta) aligns with industry best practice.

## category.5: DNS management constraints

### [FACT] A/AAAA + CNAME conflict

> "A/AAAA records cannot exist on the same name as CNAME records."

source: [10] https://developers.cloudflare.com/api/resources/dns/

**implication**: cloudflare API enforces this constraint. declastruct does not need extra validation; API rejects invalid state.

### [FACT] TTL minimums vary by plan

> "TTL... Value of 1 for this parameter means 'automatic'. Value must be between 60 and 86400, with the minimum reduced to 30 for Enterprise zones."

source: [10] https://developers.cloudflare.com/api/resources/dns/

**implication**: TTL constraints are plan-dependent. declastruct can accept user values and let API enforce limits.

## category.6: multi-provider DNS management

### [OPIN] multi-provider IaC frameworks reduce complexity

> "Tools like dnscontrol provide a unified language supporting multiple DNS providers."

source: [8] https://umatechnology.org/infrastructure-as-code-examples-for-multi-zone-dns-management-with-config-as-code/

**implication**: declastruct follows a similar pattern — unified domain-objects interface across providers (cloudflare, aws, etc.).

### [SUMP] TypeScript for IaC is more developer-friendly

> "Pulumi IaC lets you use languages like Python, Go, JavaScript, TypeScript, C#, and Java, providing familiar constructs like conditionals, loops, functions, and classes."

source: [11] https://www.pulumi.com/docs/iac/comparisons/terraform/

**implication**: declastruct's TypeScript resources.ts approach aligns with modern IaC trends. developers prefer real programming languages over DSLs.

## category.7: authentication and security

### [FACT] bearer token authentication is preferred

> "The preferred authorization scheme for interacting with the Cloudflare API is Bearer token authentication."

source: [12] https://developers.cloudflare.com/api/resources/zones/methods/create/

**implication**: provider must require apiToken, not apiKey. apiKey auth is deprecated.

### [FACT] WHOIS privacy redaction is free with cloudflare registrar

> "Cloudflare Registrar offers personal data redaction on WHOIS for free, meeting current ICANN guidelines."

source: [13] https://developers.cloudflare.com/registrar/account-options/whois-redaction/

**implication**: privacyProtection option in DeclaredCloudflareDomainRegistration is enabled by default on cloudflare.

## category.8: open questions

### [KHUE] can registrar.domains.update work without enterprise?

the community posts suggest GET-only for non-enterprise, but the official API docs show an update endpoint. need integration tests to confirm if update works with standard API tokens.

### [KHUE] what registrar-specific transfer docs should we link?

transfer guidance should link registrar-specific transfer-out docs (godaddy, route53, namecheap, etc.). which registrars should we support with specific guidance?

### [KHUE] should declastruct track transfer progress?

cloudflare API may expose transfer status fields. should setDomainRegistration return status when transfer is in progress, or throw until complete?

## summary

| category | count | key finding |
|----------|-------|-------------|
| ICANN transfer requirements | 4 FACT | 60-day lock, 5-7 day wait, auth code required |
| cloudflare registrar API | 4 FACT | registration API enterprise-only, guided approach validated |
| cloudflare WHOIS intel API | 2 FACT | can detect domain state for smart guidance |
| IaC best practices | 4 FACT/OPIN | idempotency, declarative, plan-before-apply |
| DNS constraints | 2 FACT | A/CNAME conflict, TTL limits by plan |
| multi-provider DNS | 2 SUMP/OPIN | TypeScript IaC is developer-friendly |
| authentication | 2 FACT | bearer token preferred, WHOIS privacy free |
