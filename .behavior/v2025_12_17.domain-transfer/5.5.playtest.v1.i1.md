## edgey path 1: transfer guidance

### scenario: domain registered elsewhere

when a domain is registered at another registrar (e.g., godaddy) and setDomainRegistration is called, it should throw a BadRequestError with transfer guidance.

**verification via unit test**:

```sh
npm run test:unit -- setDomainRegistration.test.ts
```

look for test case:
- `when('the registration does not exist and whois shows domain is registered elsewhere') then('it should throw a BadRequestError with transfer guidance')`

**expected outcome**: test passes, confirms transfer guidance is thrown

## edgey path 3: DnsRecord unique key includes content

### scenario: multiple A records with same name but different IPs

the unique key for DnsRecord is `['zone', 'name', 'type', 'content']` — not just `['zone', 'name', 'type']`.

this allows multiple A records for load distribution:
- api.example.com → 192.168.1.1
- api.example.com → 192.168.1.2

**verification via type check**:

```sh
npm run test:types
```

**expected outcome**: no type errors. unique lookups require `content` field.
