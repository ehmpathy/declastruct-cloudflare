# research: domain

research on domain objects and operations to fulfill the wish, vision, and criteria.

## domain objects

| object | type | source | purpose |
|--------|------|--------|---------|
| Zone | entity | cloudflare zones api | DNS container for domain |
| DnsRecord | entity | cloudflare dns api | individual DNS entry |
| Registration | entity | cloudflare registrar api | domain ownership |
| WhoisRecord | literal | cloudflare intel api | domain state detection |

## entity.1: Zone

### declastruct shape

```
DeclaredCloudflareDomainZone
├─ primary = ["id"]
├─ unique = ["name"]
├─ readonly = ["id"]
└─ properties
   ├─ id?: string (readonly, cloudflare-assigned)
   ├─ name: string (immutable)
   └─ type: "full" | "partial"
```

### properties from cloudflare api

| property | type | declastruct? |
|----------|------|--------------|
| id | string | ✓ readonly |
| name | string | ✓ unique key |
| type | enum | ✓ configurable |
| status | enum | internal only |
| account.id | string | from context |
| name_servers | string[] | read-only info |
| activated_on | timestamp | internal only |
| created_on | timestamp | internal only |

### operations

| operation | semantics | endpoint |
|-----------|-----------|----------|
| getAllDomainZones | list all | GET /zones |
| getOneDomainZone | lookup | GET /zones/{id} or filter by name |
| setDomainZone (findsert) | find or create | POST /zones |
| setDomainZone (upsert) | update | PATCH /zones/{id} |
| delDomainZone | remove | DELETE /zones/{id} |

## entity.2: DnsRecord

### declastruct shape

```
DeclaredCloudflareDomainDnsRecord
├─ primary = ["id"]
├─ unique = ["zone.name", "name", "type", "content"]
├─ readonly = ["id"]
└─ properties
   ├─ id?: string (readonly, cloudflare-assigned)
   ├─ zone: RefByUnique<Zone> (reference)
   ├─ name: string (full record name)
   ├─ type: DnsRecordType
   ├─ content: string
   ├─ ttl: number (60-86400, or 1 for auto)
   ├─ proxied?: boolean (A, AAAA, CNAME only)
   └─ priority?: number (MX, SRV only)
```

### unique key rationale

cloudflare allows multiple records with same (zone, name, type) for load distribution (e.g., multiple A records). content as part of unique key enables:
- each record independently declarable
- multiple A records for round-robin
- findsert semantics per-record

### dns record types

| type | proxied? | priority? | use case |
|------|----------|-----------|----------|
| A | yes | no | IPv4 address |
| AAAA | yes | no | IPv6 address |
| CNAME | yes | no | alias |
| MX | no | yes | mail exchange |
| TXT | no | no | text (SPF, DKIM) |
| NS | no | no | nameserver |
| SRV | no | yes | service location |
| CAA | no | no | certificate authority |

### operations

| operation | semantics | endpoint |
|-----------|-----------|----------|
| getAllDomainDnsRecords | list for zone | GET /zones/{zone_id}/dns_records |
| getOneDomainDnsRecord | lookup | GET /zones/{zone_id}/dns_records/{id} |
| setDomainDnsRecord (findsert) | find or create | POST /zones/{zone_id}/dns_records |
| setDomainDnsRecord (upsert) | update | PUT /zones/{zone_id}/dns_records/{id} |
| delDomainDnsRecord | remove | DELETE /zones/{zone_id}/dns_records/{id} |

## entity.3: Registration

### declastruct shape

```
DeclaredCloudflareDomainRegistration
├─ primary = ["id"]
├─ unique = ["name"]
├─ readonly = ["id"]
└─ properties
   ├─ id?: string (readonly, cloudflare-assigned)
   ├─ name: string (domain name, immutable)
   ├─ autoRenew: boolean
   ├─ locked: boolean
   └─ privacyProtection: boolean
```

### api-only properties (not declared)

| property | type | description |
|----------|------|-------------|
| expires_at | timestamp | expiration date |
| current_registrar | string | registrar name |
| registry_statuses | string | status codes |
| transfer_in | object | transfer progress |

### operations

| operation | semantics | automatable? |
|-----------|-----------|--------------|
| getAllDomainRegistrations | list | yes |
| getOneDomainRegistration | lookup | yes |
| setDomainRegistration (findsert) | find or throw guidance | partially |
| setDomainRegistration (upsert) | update or throw guidance | partially |
| delDomainRegistration | n/a | no (dashboard only) |

### guidance flow

```
setDomainRegistration(domain)
├─ domain in cloudflare? → findsert/upsert
└─ domain not in cloudflare?
   ├─ WHOIS found? → throw transfer guidance
   └─ WHOIS not found? → throw purchase guidance
```

## literal.1: WhoisRecord

### shape (internal use only)

```
WhoisRecord
└─ properties
   ├─ domain: string
   ├─ found: boolean
   ├─ registrar: string | null
   ├─ nameservers: string[]
   └─ dnssec: boolean
```

### operations

| operation | endpoint |
|-----------|----------|
| getWhoisRecord | GET /accounts/{account_id}/intel/whois?domain={domain} |

### use case

enables smart guidance in setDomainRegistration:
- `found: true` + registrar → transfer guidance with registrar name
- `found: false` → purchase guidance

## relationships

### hierarchy (treestruct)

```
Zone (container)
└─ DnsRecord (many)
   ├─ A records
   ├─ AAAA records
   ├─ CNAME records
   ├─ MX records
   └─ ...

Registration (ownership)
└─ Zone (required for registrar features)
```

### dependencies

| from | to | relationship |
|------|-----|--------------|
| DnsRecord | Zone | belongs to (required) |
| Registration | Zone | requires (for registrar) |
| WhoisRecord | (none) | independent lookup |

### apply order

1. zones first (container)
2. dns records second (depend on zones)
3. registrations last (may throw guidance)
