## test patterns in use

### 1. bdd structure with test-fns

```ts
import { given, then, useBeforeAll, when } from 'test-fns';

describe('domainZone', () => {
  given('a cloudflare account', () => {
    when('[t0] getAllDomainZones is called', () => {
      then('it should return an array of zones', async () => {
        const zones = await getAllDomainZones(getContext());
        expect(Array.isArray(zones)).toBe(true);
      });
    });
  });
});
```

### 2. shared setup with useBeforeAll

```ts
given('a cloudflare account with at least one zone', () => {
  const zonesData = useBeforeAll(async () => {
    const allZones = await getAllDomainZones(getContext());
    return { zones: allZones, firstZone: allZones[0] ?? null };
  });

  when('[t0] getOneDomainZone is called by primary key', () => {
    then('it should return the zone if zones exist', async () => {
      if (!zonesData.firstZone) return;
      // ...
    });
  });
});
```

### 3. cli workflow in acceptance tests

```ts
// run declastruct plan command
execSync(`npx declastruct plan --wish ${resourcesFile} --into ${planFile}`, {
  stdio: 'inherit',
  env: process.env,
  cwd: join(__dirname, '..', '..', '..'),
});

// read and parse the plan
const planContent = readFileSync(planFile, 'utf-8');
const plan = JSON.parse(planContent);
```

## recommended tools

### 1. test-fns (in use)

```ts
import { given, then, useBeforeAll, when } from 'test-fns';
```

| feature | benefit |
|---------|---------|
| given/when/then | bdd structure |
| useBeforeAll | efficient shared setup |
| useBeforeEach | fresh setup per test |

### 2. jest assertions (in use)

```ts
expect(zone).not.toBeNull();
expect(zone?.id).toEqual(zonesData.firstZone.id);
expect(Array.isArray(zones)).toBe(true);
```

| feature | benefit |
|---------|---------|
| toEqual | deep equality |
| toBeNull | null checks |
| toHaveProperty | shape assertions |

### 3. helpful-errors getError (recommended)

```ts
import { getError } from 'test-fns';

const error = await getError(setDomainRegistration({ findsert: registration }, context));
expect(error).toBeInstanceOf(BadRequestError);
expect(error.message).toContain('transfer required');
```

| feature | benefit |
|---------|---------|
| getError | capture expected errors |
| instance check | verify error type |
| message check | verify guidance content |

## test loop categories

### category.1: unit tests (fast feedback)

| what | how |
|------|-----|
| domain object shape | instantiate with test data |
| cast functions | transform sdk response → domain object |
| operation logic | mock context, verify behavior |

**run**: `npm run test:unit`

### category.2: integration tests (api verification)

| what | how |
|------|-----|
| read operations | call live api, verify response shape |
| idempotency | call twice, verify same result |
| error handling | trigger api errors, verify wrap |

**run**: `npm run test:integration`

### category.3: acceptance tests (end-to-end)

| what | how |
|------|-----|
| plan workflow | run cli, verify plan.json structure |
| apply workflow | run cli, verify no errors |
| idempotency | apply twice, verify second pass clean |

**run**: `npm run test:acceptance:locally`

## reproduction patterns

### experience: api surface

```ts
// declare resource
const zone = new DeclaredCloudflareDomainZone({ name: 'example.com', type: 'full' });

// verify shape
expect(zone.name).toBe('example.com');
expect(DeclaredCloudflareDomainZone.unique).toEqual(['name']);
```

### experience: return values

```ts
// call operation
const result = await setDomainZone({ findsert: zone }, context);

// verify return shape
expect(result.id).toBeDefined();
expect(result.name).toBe('example.com');
```

### experience: error conditions

```ts
// trigger error
const error = await getError(setDomainRegistration({ findsert: registration }, context));

// verify error type and guidance
expect(error).toBeInstanceOf(BadRequestError);
expect(error.message).toContain('transfer required');
expect(error.metadata).toHaveProperty('domain');
```
