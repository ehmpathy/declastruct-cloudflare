## 4. testing strategy

### 4.1 unit tests

- mock cloudflare client responses
- test cast functions with various SDK response shapes
- test operation logic (early returns, error handling)
- use `getMockedCloudflareApiContext` for context

### 4.2 integration tests

- use real cloudflare API (test account)
- create test zone (e.g., `test-declastruct.example.com`)
- run operations against real API
- use `getSampleCloudflareApiContext` for context
- require `CLOUDFLARE_API_TOKEN` environment variable

### 4.3 acceptance tests

- test full declastruct CLI workflow
- `declastruct plan --wish resources.ts --into plan.json`
- `declastruct apply --wish resources.ts`
- verify resources created in cloudflare

## 5. helpful error patterns

### 5.1 domain registration not found

```typescript
throw new HelpfulError({
  code: 'CLOUDFLARE_DOMAIN_REGISTRATION_REQUIRED',
  message: `domain "${name}" not found in cloudflare registrar`,
  guidance: {
    summary: 'domain registration requires manual action via cloudflare dashboard',
    steps: [
      '1. navigate to cloudflare dashboard в†’ registrar в†’ register domains',
      `2. search for domain: ${name}`,
      '3. select purchase and complete checkout',
      '4. verify domain appears in account',
      '5. re-run declastruct apply',
    ],
    link: 'https://dash.cloudflare.com/?to=/:account/domains/register',
  },
});
```

### 5.2 zone pending activation

```typescript
throw new HelpfulError({
  code: 'CLOUDFLARE_ZONE_PENDING_ACTIVATION',
  message: `zone "${name}" is pending activation`,
  guidance: {
    summary: 'zone pending activation - nameserver change required',
    steps: [
      `1. log into your domain registrar (${zone.originalRegistrar})`,
      '2. update nameservers to cloudflare\'s assigned servers:',
      ...zone.nameServers.map(ns => `     - ${ns}`),
      '3. wait for dns propagation (up to 24-48 hours)',
      '4. zone will auto-activate once verified',
      '5. re-run declastruct apply to continue',
    ],
    currentNameservers: zone.originalNameservers,
    requiredNameservers: zone.nameServers,
  },
});
```

|------|------------|
