[3] https://simpledns.plus/docs/definition-domains-vs-zones

> "The term domain is used in the business functions of the entity assigned to it, and the term zone is usually used for the configuration of DNS services."

> "A registrant is the person or company who registers a domain name."

## term analysis: zone

### extant terms

| term | source | pros | cons |
|------|--------|------|------|
| zone | cloudflare, RFC 1035 | industry standard, precise | technical |
| domain | common usage | intuitive | overloaded (means both TLD and zone) |
| site | web terminology | user-friendly | imprecise |

### recommendation

**keep "zone"** — it's the RFC standard term, precise, and cloudflare uses it consistently.

prefix: `DomainZone` — clarifies this is a zone for a domain (not firewall zone, time zone, etc.)

---

## term analysis: dns record

### extant terms

| term | source | pros | cons |
|------|--------|------|------|
| record | RFC 1035 | industry standard | generic |
| entry | common usage | intuitive | less precise |
| resource record (RR) | RFC 1035 | precise | verbose |

### recommendation

**keep "DnsRecord"** — industry standard, clear.

prefix: `DomainDnsRecord` — clarifies this is a DNS record within the domain context.

---

## term analysis: registration

### extant terms

| term | source | pros | cons |
|------|--------|------|------|
| registration | cloudflare, ICANN | industry standard | long |
| domain | common usage | short | overloaded |
| deed | real estate | evocative | unfamiliar in tech |
| title | real estate | evocative | overloaded (HTML title) |

### recommendation

**keep "Registration"** — industry standard, maps to registrar concept.

prefix: `DomainRegistration` — clarifies this is domain ownership record.

## proposed domain object names

### treestruct

```
Declared                    # prefix: declared state (declastruct pattern)
└─ Cloudflare               # provider namespace
   └─ Domain                # domain context (DNS, registrar)
      ├─ Zone               # entity: DNS zone container
      ├─ DnsRecord          # entity: individual DNS record
      └─ Registration       # entity: registrar-level ownership
```

### final names

| object | name | rationale |
|--------|------|-----------|
| zone | `DeclaredCloudflareDomainZone` | RFC standard, cloudflare native |
| dns record | `DeclaredCloudflareDomainDnsRecord` | RFC standard, explicit context |
| registration | `DeclaredCloudflareDomainRegistration` | industry standard, clear intent |

### type aliases

for convenience:
```ts
type Zone = DeclaredCloudflareDomainZone;
type DnsRecord = DeclaredCloudflareDomainDnsRecord;
type Registration = DeclaredCloudflareDomainRegistration;
```

---

## rejected alternatives

| rejected | reason |
|----------|--------|
| `CloudflareSite` | imprecise, web-centric |
| `CloudflareDomain` | overloaded (zone vs registration) |
| `DomainDeed` | unfamiliar, breaks industry convention |
| `DnsZone` | redundant (zone is already DNS concept) |
| `DomainEntry` | less precise than DnsRecord |

## relationships

### dependencies

```
DnsRecord
└─ requires → Zone (via RefByUnique)

Registration
└─ related to → Zone (same domain name, different concerns)
```

### treestruct of subdomains

```
example.com (Zone)
├─ api.example.com (DnsRecord: A)
├─ www.example.com (DnsRecord: CNAME)
├─ mail.example.com (DnsRecord: MX)
└─ *.example.com (DnsRecord: A, wildcard)
```

## summary

| object | final name | key rationale |
|--------|------------|---------------|
| zone | DeclaredCloudflareDomainZone | RFC 1035 standard |
| dns record | DeclaredCloudflareDomainDnsRecord | clear |
| registration | DeclaredCloudflareDomainRegistration | registrar concept |

all names follow the pattern: `Declared{Provider}{Context}{Concept}`
