## object.zone

### citations

[1] https://developers.cloudflare.com/api/resources/zones/

> zones api provides: create, list, get, edit, delete operations

[2] https://developers.cloudflare.com/api/resources/zones/methods/list/

> "List Zones" — lists, searches, sorts, and filters your zones

[3] https://registry.terraform.io/providers/cloudflare/cloudflare/latest/docs/resources/zone

> "A full zone implies that DNS is hosted with Cloudflare. A partial zone is typically a partner-hosted zone or a CNAME setup."

> "Available zone types are: 'full', 'partial', 'secondary', 'internal'."

[4] https://developers.cloudflare.com/api/resources/zones/methods/get/

> "Zone Details" endpoint returns zone properties

### properties

| property | type | description |
|----------|------|-------------|
| id | string | unique identifier (e.g., "023e105f4ecef8ad9ca31a8372d0c353") |
| name | string | domain name (up to 253 characters per RFC 1035) |
| type | enum | "full", "partial", "secondary", "internal" |
| status | enum | "initializing", "pending", "active", "moved" |
| account.id | string | account identifier |
| account.name | string | account name |
| name_servers | string[] | cloudflare-assigned nameservers |
| activated_on | timestamp | when zone became active |
| created_on | timestamp | when zone was created |
| modified_on | timestamp | when zone was last modified |

### unique key

name (domain name)

### primary key

id (cloudflare-assigned)

### declastruct shape

```
DeclaredCloudflareDomainZone
├─ primary = ["id"]
├─ unique = ["name"]
└─ properties
   ├─ id?: string (readonly, cloudflare-assigned)
   ├─ name: string (immutable)
   └─ type: "full" | "partial"
```

## object.dnsrecord

### citations

[5] https://developers.cloudflare.com/api/resources/dns/subresources/records/

> DNS records api with various record types

[6] https://developers.cloudflare.com/api/resources/dns/subresources/records/methods/create/

> "Create a new DNS record for a zone."

[7] https://developers.cloudflare.com/api/resources/dns/

> "TTL... Value must be between 60 and 86400, with the minimum reduced to 30 for Enterprise zones."

> "A/AAAA records cannot exist on the same name as CNAME records."

> "NS records cannot exist on the same name as any other record type."

> "Domain names are always represented in Punycode."

### properties

| property | type | description |
|----------|------|-------------|
| id | string | unique record identifier |
| zone_id | string | parent zone identifier |
| zone_name | string | parent zone name |
| name | string | record name (full, e.g., "api.example.com") |
| type | enum | A, AAAA, CNAME, MX, TXT, NS, SRV, CAA, etc. |
| content | string | record value (IP, hostname, text) |
| ttl | number | time to live (60-86400, or 1 for automatic) |
| proxied | boolean | whether cloudflare proxy is enabled |
| priority | number | MX/SRV priority (optional) |
| tags | string[] | custom tags (name:value format) |
| comment | string | optional comment |
| created_on | timestamp | when record was created |
| modified_on | timestamp | when record was last modified |

### unique key

**design decision needed**: cloudflare allows multiple records with same (zone_id, name, type). for example, multiple A records for load distribution.

options:
1. unique = ["zone.name", "name", "type", "content"] — includes content to differentiate
2. unique = ["zone.name", "name", "type"] — requires special multi-record logic

### primary key

id (cloudflare-assigned)

### declastruct shape

```
DeclaredCloudflareDomainDnsRecord
├─ primary = ["id"]
├─ unique = ["zone.name", "name", "type", "content"]
└─ properties
   ├─ id?: string (readonly, cloudflare-assigned)
   ├─ zone: RefByUnique<Zone> (reference to parent zone)
   ├─ name: string (full record name)
   ├─ type: DnsRecordType
   ├─ content: string
   ├─ ttl: number (60-86400, or 1 for automatic)
   ├─ proxied?: boolean (A, AAAA, CNAME only)
   └─ priority?: number (MX, SRV only)
```

## relationships

### treestruct (hierarchy)

```
Zone (container)
└─ DnsRecord (many)
   ├─ A record
   ├─ AAAA record
   ├─ CNAME record
   ├─ MX record
   └─ ...

Registration (ownership)
└─ Zone (required for registrar features)
```

### dependencies

1. **DnsRecord depends on Zone**: records belong to zones; zone must exist before records
2. **Registration requires Zone**: domain must be added as zone to use cloudflare registrar
3. **WhoisRecord is independent**: lookup-only, no persistence

### composition for wish fulfillment

```
wish: declarative control of domains via declastruct

fulfillment:
1. declare Zone → creates/manages DNS container
2. declare DnsRecord (via Zone ref) → creates/manages DNS entries
3. declare Registration (via Zone ref) → manages registrar settings OR triggers guidance

workflow:
  ┌─────────────┐
  │ resources.ts │
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ plan.json   │ ← compare declared vs actual
  └──────┬──────┘
         │
         ▼
  ┌─────────────────────────────────────────────┐
  │ apply                                       │
  │ 1. zones first (dependency)                 │
  │ 2. dns records second                       │
  │ 3. registrations last (may throw guidance)  │
  └─────────────────────────────────────────────┘
```
