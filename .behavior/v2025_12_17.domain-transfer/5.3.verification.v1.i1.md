| domainWhoisRecord.integration.test.ts:19 | describe.skip | credential-based skip | ✓ yes |
| index.integration.test.ts:30 | describe.skip | credential-based skip | ✓ yes |

**why credential-based skips are acceptable**:
- integration tests require live cloudflare API access
- credentials may not be available in all CI environments
- pattern: `const hasCredentials = ...; const describeWithCredentials = hasCredentials ? describe : describe.skip;`
- tests run fully when credentials are provided; skip gracefully when not
- this is not a bypass of test coverage — it is conditional execution based on environment

### silent bypass scan

- no hardcoded `return` or `if (false)` patterns found
- no `process.exit` in test files
- no `jest.fn()` mocks that short-circuit actual behavior
