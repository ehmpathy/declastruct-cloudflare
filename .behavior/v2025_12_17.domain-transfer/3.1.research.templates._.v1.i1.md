# research: templates

research on declastruct ecosystem templates to leverage for declastruct-cloudflare.

## objective

identify key patterns from established declastruct providers that we should follow.

## template.1: declastruct-github

### citations

[1] https://github.com/ehmpathy/declastruct-github

> "declarative control of GitHub constructs via declastruct - manage repos, branches, and protection rules"

[2] https://raw.githubusercontent.com/ehmpathy/declastruct-github/main/readme.md

> "Resources are declared via a fluent .as() method"

> "The getResources() function returns an array of all declared entities that form the desired state"

### domain objects

| object | purpose | unique key |
|--------|---------|------------|
| DeclaredGithubRepo | repository | [owner, name] |
| DeclaredGithubRepoConfig | repo settings | [repo.owner, repo.name] |
| DeclaredGithubBranch | branch | [name, repo.owner, repo.name] |
| DeclaredGithubBranchProtection | protection rules | [branch.name, branch.repo] |

### key patterns

#### 1. reference pattern

```ts
const branchMain = RefByUnique.as<typeof DeclaredGithubBranch>({
  name: 'main',
  repo,  // instance passed directly
});
```

#### 2. provider setup pattern

```ts
getDeclastructGithubProvider({
  credentials: { token: process.env.GITHUB_TOKEN },
}, { log: console })
```

#### 3. getResources pattern

```ts
export const getResources = async () => {
  const repo = DeclaredGithubRepo.as({ ... });
  const config = DeclaredGithubRepoConfig.as({ repo, ... });
  return [repo, config];
};
```

### relevance to wish

| github pattern | cloudflare parallel |
|----------------|---------------------|
| DeclaredGithubRepo | DeclaredCloudflareDomainZone |
| DeclaredGithubBranch | DeclaredCloudflareDomainDnsRecord |
| DeclaredGithubRepoConfig | DeclaredCloudflareDomainRegistration |
| repo → branch refs | zone → dns record refs |

## template.2: declastruct-stripe-sdk

### citations

[3] https://github.com/ehmpathy/declastruct-stripe-sdk

> "declarative control of stripe constructs, via declastruct"

[4] https://raw.githubusercontent.com/ehmpathy/declastruct-stripe-sdk/main/readme.md

> "Declare the structures you want. Plan to see the changes required. Apply to make it so."

### domain objects

| object | purpose | unique key |
|--------|---------|------------|
| DeclaredStripeProduct | product | [name] |
| DeclaredStripeCustomer | customer | [email] |
| DeclaredStripeInvoice | invoice | [customer, number] |
| DeclaredStripeInvoiceItem | line item | [invoice, description] |

### key patterns

#### 1. set operations pattern

```ts
export const setProduct = async (
  input: PickOne<{
    findsert: DeclaredStripeProduct;
    upsert: DeclaredStripeProduct;
  }>,
  context: StripeApiContext & VisualogicContext,
): Promise<HasMetadata<DeclaredStripeProduct>> => {
  // find extant
  const found = await getProduct({ by: { unique: { ... } } }, context);

  // findsert: return if found
  if (input.findsert && found) return found;

  // upsert: update if found
  if (input.upsert && found) {
    return castToDeclaredStripeProduct(
      await context.stripe.products.update(found.id, { ... }),
    );
  }

  // create if not found
  return castToDeclaredStripeProduct(
    await context.stripe.products.create({ ... }),
  );
};
```

#### 2. cast function pattern

```ts
export const castToDeclaredStripeCustomer = (
  input: Stripe.Customer,
): HasMetadata<DeclaredStripeCustomer> => {
  return new DeclaredStripeCustomer({
    id: input.id,
    email: input.email ?? UnexpectedCodePathError.throw('...'),
    name: input.name ?? null,
    // ...
  }) as HasMetadata<DeclaredStripeCustomer>;
};
```

#### 3. context pattern

```ts
export type StripeApiContext = {
  stripe: Stripe;
};

const context: StripeApiContext & VisualogicContext = {
  stripe: await getStripe(credentials),
  log: console,
};
```

### relevance to wish

| stripe pattern | cloudflare parallel |
|----------------|---------------------|
| StripeApiContext | ContextCloudflareApi |
| castToDeclaredStripeCustomer | castIntoDeclaredCloudflareDomainZone |
| setProduct with findsert/upsert | setDomainZone with findsert/upsert |

## template.3: declastruct (core)

### citations

[5] https://github.com/ehmpathy/declastruct

> "Add declarative control to any resource constructs. Declare, plan, and apply within an observable pit-of-success."

[6] https://www.npmjs.com/package/declastruct

> "bundles related DAOs and provider context"

### provider interface

```ts
interface DeclastructProvider<T> {
  name: string;
  daos: Record<string, DeclastructDao>;
  context: T;
  hooks: {
    beforeAll: () => Promise<void>;
    afterAll: () => Promise<void>;
  };
}
```

### dao interface

```ts
interface DeclastructDao<T> {
  dobj: T;
  get: {
    one: {
      byPrimary?: (input, context) => Promise<T | null>;
      byUnique?: (input, context) => Promise<T | null>;
    };
  };
  set: {
    findsert?: (input, context) => Promise<T>;
    upsert?: (input, context) => Promise<T>;
    delete?: (input, context) => Promise<void>;
  };
}
```

### genDeclastructDao pattern

```ts
export const DeclaredCloudflareDomainZoneDao = genDeclastructDao<
  typeof DeclaredCloudflareDomainZone,
  ContextCloudflareApi & ContextLogTrail
>({
  dobj: DeclaredCloudflareDomainZone,
  get: {
    one: {
      byPrimary: async (input, context) => { ... },
      byUnique: async (input, context) => { ... },
    },
  },
  set: {
    findsert: async (input, context) => { ... },
    upsert: async (input, context) => { ... },
    delete: async (input, context) => { ... },
  },
});
```

### relevance to wish

- use genDeclastructDao for all resource DAOs
- implement byPrimary, byUnique for lookup
- implement findsert, upsert, delete for mutations
- wrap in DeclastructProvider with name, context, hooks

## template.4: declastruct-cloudflare (extant)

### citations

[7] src/domain.operations/provider/getDeclastructCloudflareProvider.ts

```ts
export const getDeclastructCloudflareProvider = async (
  input: { apiToken?: string; accountId?: string },
  context: ContextLogTrail,
): Promise<DeclastructCloudflareProvider> => {
  const credentials = getCredentials(input);
  const client = new Cloudflare({ apiToken: credentials.apiToken });
  const providerContext = { ...context, cloudflare: { client, accountId } };
  const daos = {
    DeclaredCloudflareDomainZone: DeclaredCloudflareDomainZoneDao,
    DeclaredCloudflareDomainDnsRecord: DeclaredCloudflareDomainDnsRecordDao,
    DeclaredCloudflareDomainRegistration: DeclaredCloudflareDomainRegistrationDao,
  };
  return new DeclastructProvider({ name: 'cloudflare', daos, context: providerContext, hooks: { ... } });
};
```

[8] src/access/daos/DeclaredCloudflareDomainZoneDao.ts

```ts
export const DeclaredCloudflareDomainZoneDao = genDeclastructDao<
  typeof DeclaredCloudflareDomainZone,
  ContextCloudflareApi & ContextLogTrail
>({
  dobj: DeclaredCloudflareDomainZone,
  get: { one: { byPrimary, byUnique } },
  set: { findsert, upsert, delete },
});
```

### patterns established

- provider follows getDeclastructXxxProvider name convention
- context composes cloudflare client + accountId
- daos map domain object names to genDeclastructDao wrappers
- operations follow getOne/set/del name convention

## patterns to leverage

### 1. domain object declaration

```ts
interface DeclaredCloudflareDomainZone {
  id?: string;
  name: string;
  type: 'full' | 'partial';
}

class DeclaredCloudflareDomainZone
  extends DomainEntity<DeclaredCloudflareDomainZone>
  implements DeclaredCloudflareDomainZone
{
  public static primary = ['id'] as const;
  public static unique = ['name'] as const;
}
```

### 2. cast function

```ts
export const castIntoDeclaredCloudflareDomainZone = (
  input: Cloudflare.Zone,
): HasMetadata<DeclaredCloudflareDomainZone> => {
  return new DeclaredCloudflareDomainZone({
    id: input.id,
    name: input.name,
    type: input.type,
  }) as HasMetadata<DeclaredCloudflareDomainZone>;
};
```

### 3. operation pattern

```ts
export const setDomainZone = async (
  input: PickOne<{
    findsert: DeclaredCloudflareDomainZone;
    upsert: DeclaredCloudflareDomainZone;
  }>,
  context: ContextCloudflareApi & ContextLogTrail,
): Promise<HasMetadata<DeclaredCloudflareDomainZone>> => {
  // lookup extant
  const found = await getOneDomainZone({ by: { unique: { ... } } }, context);

  // findsert: return if found
  if (input.findsert && found) return found;

  // upsert: update if found
  if (input.upsert && found) { ... }

  // create
  return castIntoDeclaredCloudflareDomainZone(
    await context.cloudflare.client.zones.create({ ... }),
  );
};
```

### 4. dao factory

```ts
export const DeclaredCloudflareDomainZoneDao = genDeclastructDao<
  typeof DeclaredCloudflareDomainZone,

  dobj: DeclaredCloudflareDomainZone,
```

### 5. provider assembly

```ts
export const getDeclastructCloudflareProvider = async (
  input: { apiToken?: string; accountId?: string },

  const client = new Cloudflare({ apiToken });
  return new DeclastructProvider({
```

## summary

| template | key contribution |
|----------|------------------|
| declastruct-github | reference pattern, getResources structure |
| declastruct-stripe-sdk | cast functions, findsert/upsert operations |
| declastruct (core) | genDeclastructDao, provider interface |
| declastruct-cloudflare (extant) | cloudflare-specific context, name conventions |

all patterns are consistent across templates. declastruct-cloudflare implementation should follow these established patterns.
