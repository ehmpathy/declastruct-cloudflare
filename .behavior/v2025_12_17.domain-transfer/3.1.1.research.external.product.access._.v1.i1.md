## lesson.1: cloudflare official node.js SDK

### summary

cloudflare provides an official TypeScript SDK for node.js, published as the `cloudflare` npm package.

### citations

[1] https://www.npmjs.com/package/cloudflare

> "The official TypeScript library for the Cloudflare API."

> "Node.js 18 LTS or later (non-EOL) versions are supported."

> "TypeScript >= 4.5 is supported."

[2] https://github.com/cloudflare/cloudflare-typescript

> "cloudflare is archiving the older repository (cloudflare/node-cloudflare) in favour of the newer repository (cloudflare/cloudflare-typescript)"

### lesson

use `cloudflare` npm package (v5.x+) from `cloudflare/cloudflare-typescript` repository. older `node-cloudflare` repository is archived.

## lesson.2: zones api contract

### summary

cloudflare zones api supports full CRUD with REST endpoints.

### citations

[3] https://developers.cloudflare.com/api/resources/zones/methods/create/

> "POST https://api.cloudflare.com/client/v4/zones"

> "Request body: { 'account': {}, 'name': 'example.com', 'type': 'full' }"

[4] https://developers.cloudflare.com/api/resources/zones/methods/list/

> "List, search, sort, and filter your zones."

[5] https://developers.cloudflare.com/api/resources/zones/

> zones api provides: create, list, get, edit, delete operations

### lesson

zones api supports:
- POST /zones → create zone
- GET /zones → list zones
- GET /zones/{zone_id} → get zone details
- PATCH /zones/{zone_id} → edit zone
- DELETE /zones/{zone_id} → delete zone

authentication via Bearer token (api token).

## lesson.3: dns records api contract

### summary

cloudflare dns api supports full CRUD for dns records within zones.

### citations

[6] https://developers.cloudflare.com/api/resources/dns/

> "DNS" api with subresources for records

[7] https://developers.cloudflare.com/api/resources/dns/subresources/records/methods/create/

> "Create a new DNS record for a zone."

[8] https://developers.cloudflare.com/api/resources/dns/subresources/records/methods/list/

> "List, search, sort, and filter a zone's DNS records."

[9] https://developers.cloudflare.com/api/resources/dns/subresources/records/methods/update/

> "Overwrite DNS Record"

[10] https://developers.cloudflare.com/api/resources/dns/

> "TTL... Setting of 1 for this parameter means 'automatic'. Value must be between 60 and 86400, with the minimum reduced to 30 for Enterprise zones."

> "A/AAAA records cannot exist on the same name as CNAME records. NS records cannot exist on the same name as any other record type."

### lesson

dns records api supports:
- POST /zones/{zone_id}/dns_records → create record
- GET /zones/{zone_id}/dns_records → list records
- GET /zones/{zone_id}/dns_records/{record_id} → get record
- PUT /zones/{zone_id}/dns_records/{record_id} → overwrite record
- PATCH /zones/{zone_id}/dns_records/{record_id} → patch record
- DELETE /zones/{zone_id}/dns_records/{record_id} → delete record

constraints enforced by api:
- A/AAAA + CNAME conflict rejected
- TTL minimum 60s (30s for enterprise)
- proxied only valid for A, AAAA, CNAME

## lesson.4: registrar api contract

### summary

cloudflare registrar api supports list and update operations. domain registration and transfer are NOT automatable via api — dashboard required.

### citations

[11] https://developers.cloudflare.com/api/resources/registrar/subresources/domains/methods/list/

> "List Domains" — lists domains registered with cloudflare registrar

[12] https://developers.cloudflare.com/api/resources/registrar/subresources/domains/methods/update/

> "PUT /accounts/{account_id}/registrar/domains/{domain_name}"

> "auto_renew: Optional; Controls whether a subscription is automatically renewed upon domain expiration."

> "locked: Optional; Shows whether a registrar lock is in place for a domain."

> "privacy: Optional; Shows the owner contact information, redacting WHOIS contact information."

[13] https://community.cloudflare.com/t/register-and-manage-domains-with-cloudflare-registrar-api/453793

> "API for domain registration is available but only for Enterprise customers"

[14] https://community.cloudflare.com/t/domain-registration-using-the-api/382556

> domain registration via api has limited functionality compared to dashboard

### lesson

registrar api supports:
- GET /accounts/{account_id}/registrar/domains → list domains
- PUT /accounts/{account_id}/registrar/domains/{domain_name} → update settings

settings that can be updated: auto_renew, locked, privacy

**critical limitation**: domain registration and domain transfer are NOT automatable via api. users must use cloudflare dashboard. this validates the "guided" approach in vision.

## lesson.5: whois intel api for domain detection

### summary

cloudflare intel api provides whois lookup to detect domain ownership — enables smart guidance (transfer vs purchase).

### citations

[15] https://developers.cloudflare.com/api/resources/intel/subresources/whois/methods/get/

> "GET /accounts/{account_id}/intel/whois?domain={domain}"

> "Response includes: dnssec, domain, extension, found, nameservers, punycode, registrant, registrar"

[16] https://developers.cloudflare.com/api/resources/intel/subresources/whois/

> "Get WHOIS Record"

### lesson

whois api returns:
- `found: boolean` — whether domain is registered
- `registrar: string` — current registrar name
- `nameservers: string[]` — current nameservers
- `registrant: object` — registrant info (if available)

use case: when setDomainRegistration is called for domain not in cloudflare:
1. call whois api to check if domain is registered elsewhere
2. if `found: true` → throw transfer guidance with registrar name
3. if `found: false` → throw purchase guidance

## lesson.6: domain transfer process

### summary

cloudflare domain transfer requires manual steps due to ICANN requirements. api cannot automate transfer initiation.

### citations

[17] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

> "Transferring a domain moves your registration from your current registrar to Cloudflare."

> "The active work takes about 30 minutes. However, the total time is around 5 business days."

> "Your domain must have been registered at least 60 days ago and not transferred in the last 60 days."

> "Before you can transfer, you must first add your domain to Cloudflare."

[18] https://developers.cloudflare.com/registrar/faq/

> ICANN requires human verification steps (FOA email)

### lesson

transfer process (manual steps required):
1. add domain to cloudflare as zone (automatable)
2. at current registrar: disable dnssec, disable whois privacy, unlock domain, get auth code
3. at cloudflare dashboard: initiate transfer with auth code
4. approve FOA email
5. wait up to 5 days (ICANN requirement)
6. re-run declastruct apply

**this validates the HelpfulError guidance approach** — we cannot automate transfer, but we can provide step-by-step guidance.

## lesson.7: authentication best practices

### summary

api token authentication is preferred. api key authentication is deprecated.

### citations

[19] https://developers.cloudflare.com/api/resources/zones/methods/create/

> "The preferred authorization scheme for interacting with the Cloudflare API is Bearer token authentication."

[20] https://developers.cloudflare.com/fundamentals/api/reference/deprecations/

> api deprecations documentation — indicates api key auth moving to token-based

### lesson

use api token (Bearer token) for authentication, not api key. format:
```
Authorization: Bearer <API_TOKEN>
```

provider contract should require `apiToken`, not `apiKey`.

## lesson.8: sdk usage pattern

### summary

the cloudflare npm package provides typed methods for all api operations.

### citations

[21] https://developers.cloudflare.com/api/node/resources/dns/subresources/records/methods/create/

> node.js SDK examples for dns record operations

[22] https://www.npmjs.com/package/cloudflare

> SDK installation: `npm i cloudflare`

### lesson

sdk usage pattern:
```ts
import Cloudflare from 'cloudflare';

const client = new Cloudflare({
  apiToken: process.env.CLOUDFLARE_API_TOKEN,
});

// zones
const zones = await client.zones.list();
await client.zones.create({ account: { id: accountId }, name: 'example.com', type: 'full' });

// dns records
const records = await client.dns.records.list({ zone_id: zoneId });
await client.dns.records.create({ zone_id: zoneId, type: 'A', name: 'api', content: '192.168.1.1' });

// registrar
const domains = await client.registrar.domains.list({ account_id: accountId });
await client.registrar.domains.update(domainName, { account_id: accountId, auto_renew: true });

// intel whois
const whois = await client.intel.whois.get({ account_id: accountId, domain: 'example.com' });
```

## summary of access requirements

| api | endpoint base | auth | automatable? |
|-----|---------------|------|--------------|
| zones | /zones | Bearer token | full CRUD |
| dns records | /zones/{zone_id}/dns_records | Bearer token | full CRUD |
| registrar list | /accounts/{account_id}/registrar/domains | Bearer token | read only |
| registrar update | /accounts/{account_id}/registrar/domains/{domain} | Bearer token | update settings |
| registrar register | n/a | dashboard | guided only |
| registrar transfer | n/a | dashboard | guided only |
| intel whois | /accounts/{account_id}/intel/whois | Bearer token | read |

**key insight**: registration and transfer require dashboard. whois api enables smart detection of domain state to provide appropriate guidance.
