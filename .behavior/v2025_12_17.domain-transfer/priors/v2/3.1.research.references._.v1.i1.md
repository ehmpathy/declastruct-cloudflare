# research.references: declastruct-cloudflare knowledge base

## overview

this document consolidates authoritative references for declastruct-cloudflare domain management implementation.

## category.1: cloudflare official documentation

### ref.1.1: cloudflare zones api

**url**: https://developers.cloudflare.com/api/resources/zones/

**concepts**:
- zone creation, retrieval, update, deletion
- zone status lifecycle
- account association

**key quote** [1]:
> "Zones are the primary unit of configuration for your domain's DNS and traffic on Cloudflare"

### ref.1.2: cloudflare dns records api

**url**: https://developers.cloudflare.com/api/resources/dns/

**concepts**:
- record types: A, AAAA, CNAME, MX, TXT, NS, SRV
- proxied vs non-proxied records
- TTL constraints

**key quotes** [2]:
> "A/AAAA records cannot coexist with CNAME records on the same name"
> "NS records cannot share names with other record types"
> "TTL values must fall between 60 and 86,400 seconds"

### ref.1.3: cloudflare registrar api

**url**: https://developers.cloudflare.com/api/resources/registrar/

**concepts**:
- domain list and retrieval
- domain settings update
- transfer status

**key quote** [3]:
> "Cloudflare Registrar only supports transfers of domains that are active on a Cloudflare full setup"

### ref.1.4: cloudflare intel whois api

**url**: https://developers.cloudflare.com/api/resources/intel/subresources/whois/

**concepts**:
- domain registration status lookup
- current registrar identification
- expiration date retrieval

**usage**: detect if domain is registered elsewhere vs unregistered

### ref.1.5: cloudflare zone status reference

**url**: https://developers.cloudflare.com/dns/zone-setups/reference/domain-status/

**concepts**:
- zone statuses: init, queued, active, moved, deleted, purged
- activation requirements
- auto-deletion policy

**key quotes** [4]:
> "Queued zones cannot proxy traffic through Cloudflare"
> "Free plans auto-delete after 28 days if unactivated"

## category.3: icann policy references

### ref.3.1: icann transfer policy

**url**: https://www.icann.org/resources/pages/name-holder-faqs-2017-10-10-en

**concepts**:
- 60-day transfer lock after registration
- 60-day transfer lock after registrant change
- FOA (form of authorization) email process

**key quotes** [6]:
> "ICANN's transfer policy prohibits inter-registrar transfers within 60 days of initial registration"
> "registrars must impose a lock that will prevent any transfer to another registrar for sixty (60) days"
> "ICANN does not place, nor can it lift a 60-day lock on domain names"

## category.4: dns terminology references

### ref.4.1: dns zone definition

**url**: https://en.wikipedia.org/wiki/DNS_zone

**concepts**:
- zone as administrative boundary
- dns namespace partitions
- authoritative nameservers

**key quote** [7]:
> "A DNS zone is a specific portion of the DNS namespace in the Domain Name System (DNS), which a specific organization or administrator manages"

### ref.4.2: registrar/registry/registrant

**url**: https://directnic.com/knowledge/article/773

**concepts**:
- registrant = domain owner
- registrar = authorized seller
- registry = TLD database

**key quotes** [8]:
> "A domain name registrant is an individual or entity who registers a domain name"
> "registrars are organizations accredited by ICANN to sell domain names"

### ref.4.3: deed vs title analogy

**url**: https://www.rocketmortgage.com/learn/deed-vs-title

**concepts**:
- registration ≈ title (conceptual ownership)
- zone ≈ deed (operational control)

**key quote** [9]:
> "A deed is the physical document that passes those ownership rights from seller to buyer"
