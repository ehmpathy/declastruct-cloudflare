# research.domain.terms: declastruct-cloudflare domain management

## terminology research

### dns zone

**industry definition** [1]:
> "A DNS zone is a specific portion of the DNS namespace in the Domain Name System (DNS), which a specific organization or administrator manages."

**cloudflare definition** [2]:
> "A DNS zone is an administrative space allowing more granular control of the DNS components, such as authoritative nameserver."

**etymology**: "zone" derives from Latin *zona* ("belt, girdle") — a bounded area within a larger space. zones were named in 1983 when DNS was created [1].

---

### registrar / registry / registrant

**icann definitions** [3]:
- **registrant**: "an individual or entity who registers a domain name"
- **registrar**: "organizations accredited by ICANN... to sell domain names"
- **registry**: "database of all domain names registered under a certain TLD"

**relationship**:
```
Registry (database)
  └── Registrar (authorized seller)
        └── Registrant (domain owner)
```

---

### real estate analogy

**parallel concepts** [4]:
| dns concept | real estate concept | definition |
|-------------|---------------------|------------|
| domain name | property address | unique identifier |
| registration | title | legal ownership rights |
| registrar | title company | facilitates transfer |
| zone | deed | physical record of ownership |
| transfer | close | formal ownership change |

**key insight** [4]:
> "A deed is the physical document that passes those ownership rights from seller to buyer"
> "A title is the concept or theory of ownership that describes the person who can use or benefit from the home"

registration ≈ title (conceptual ownership)
zone ≈ deed (operational control)

## term options for domain objects

### zone entity

**current industry terms**: zone, hosted zone (aws), managed zone (google)

**options**:

| option | pros | cons |
|--------|------|------|
| `DeclaredCloudflareDomainZone` | matches cloudflare terminology exactly | long |
| `DeclaredCloudflareZone` | shorter | loses "domain" context |
| `DeclaredCloudflareDnsZone` | explicit dns scope | redundant (zones are dns) |

**recommendation**: `DeclaredCloudflareDomainZone`
- "Domain" groups zone, record, registration under common namespace
- matches cloudflare's documentation language
- "Zone" is universally understood dns terminology

---

### dns record entity

**current industry terms**: dns record, resource record (RR), record

**options**:

| option | pros | cons |
|--------|------|------|
| `DeclaredCloudflareDomainDnsRecord` | complete context | verbose |
| `DeclaredCloudflareDnsRecord` | shorter, clear scope | loses "domain" namespace |
| `DeclaredCloudflareDomainRecord` | balanced length | "record" alone is ambiguous |

**recommendation**: `DeclaredCloudflareDomainDnsRecord`
- "DnsRecord" removes ambiguity (vs database record, log record)
- "Domain" namespace groups with zone and registration
- consistent treestruct: `DeclaredCloudflare.Domain.{Zone,DnsRecord,Registration}`

---

### registration entity

**current industry terms**: domain registration, domain, registered domain

**options**:

| option | pros | cons |
|--------|------|------|
| `DeclaredCloudflareDomainRegistration` | matches icann terminology | long |
| `DeclaredCloudflareRegistration` | shorter | loses "domain" scope |
| `DeclaredCloudflareDomainTitle` | uses real estate analogy | unfamiliar in dns context |

**recommendation**: `DeclaredCloudflareDomainRegistration`
- "Registration" is exact icann/industry term
- "Domain" namespace groups with zone and dnsRecord
- avoids confusion with cloudflare "account registration"

---

### whois lookup (literal, not entity)

**current industry terms**: whois record, whois lookup, whois data

**options**:

| option | pros | cons |
|--------|------|------|
| `CloudflareWhoisRecord` | matches api response shape | "record" suggests entity |
| `CloudflareWhoisLookup` | implies read-only operation | not a stored object |
| `CloudflareIntelWhois` | matches cloudflare api path | "intel" is product name |

**recommendation**: `CloudflareWhoisRecord` (as DomainLiteral)
- used only for read-only lookups to detect transfer/purchase scenarios
- not a declared resource, so no "Declared" prefix
- literal (value object), not entity (no identity/lifecycle)

## treestruct summary

```
DeclaredCloudflare
└── Domain
    ├── Zone          // dns container (≈ deed)
    ├── DnsRecord     // individual dns entry
    └── Registration  // ownership record (≈ title)

Cloudflare
└── WhoisRecord       // read-only lookup (literal)
```
