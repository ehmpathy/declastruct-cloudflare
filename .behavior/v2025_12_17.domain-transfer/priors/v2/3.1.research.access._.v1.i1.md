# research.access: declastruct-cloudflare domain management

## remote repositories

### 1. cloudflare zones api

**contract**: REST API via `https://api.cloudflare.com/client/v4/zones`

**endpoints**:

| operation | method | path | citation |
|-----------|--------|------|----------|
| list zones | GET | `/zones` | [1] |
| get zone | GET | `/zones/{zone_id}` | [1] |
| create zone | POST | `/zones` | [1] |
| edit zone | PATCH | `/zones/{zone_id}` | [1] |
| delete zone | DELETE | `/zones/{zone_id}` | [1] |

**constraints**:
- "Listing zones across more than 500 accounts is currently not allowed" [1]
- zone types: full, partial, secondary [1]

### 2. cloudflare dns records api

**contract**: REST API via `https://api.cloudflare.com/client/v4/zones/{zone_id}/dns_records`

**endpoints**:

| operation | method | path | citation |
|-----------|--------|------|----------|
| list records | GET | `/zones/{zone_id}/dns_records` | [2] |
| get record | GET | `/zones/{zone_id}/dns_records/{dns_record_id}` | [2] |
| create record | POST | `/zones/{zone_id}/dns_records` | [2] |
| update record (full) | PUT | `/zones/{zone_id}/dns_records/{dns_record_id}` | [2] |
| update record (partial) | PATCH | `/zones/{zone_id}/dns_records/{dns_record_id}` | [2] |
| delete record | DELETE | `/zones/{zone_id}/dns_records/{dns_record_id}` | [2] |

**constraints**:
- "A/AAAA records cannot coexist with CNAME records on the same name" [2]
- "NS records cannot share names with other record types" [2]
- "TTL values must fall between 60 and 86,400 seconds. Setting TTL to 1 enables automatic mode. Enterprise zones support minimum values as low as 30 seconds." [2]

**supported record types**: A, AAAA, CAA, CNAME, DS, HTTPS, LOC, MX, NAPTR, NS, PTR, SRV, SSHFP, SVCB, TLSA, TXT, URI [2]

### 3. cloudflare registrar domains api

**contract**: REST API via `https://api.cloudflare.com/client/v4/accounts/{account_id}/registrar/domains`

**endpoints**:

| operation | method | path | citation |
|-----------|--------|------|----------|
| list domains | GET | `/accounts/{account_id}/registrar/domains` | [3] |

**response fields per domain**: id, available, can_register, created_at, current_registrar, expires_at, locked, registrant_contact, registry_statuses, supported_tld, transfer_in, updated_at [3]

**constraints**:
- "Cloudflare Registrar only supports transfers of domains that are active on a Cloudflare full setup" [4]
- "ICANN rules prohibit a domain from being transferred if the domain has been transferred within the last 60 days, the domain was registered within the last 60 days, or if the WHOIS registrant information has been modified in the last 60 days" [4]

### 4. cloudflare whois api (intel)

**contract**: REST API via `https://api.cloudflare.com/client/v4/accounts/{account_id}/intel/whois`

**endpoint**:

| operation | method | path | citation |
|-----------|--------|------|----------|
| get whois | GET | `/accounts/{account_id}/intel/whois?domain={domain}` | [5] |

**required permissions**: "Intel Write" or "Intel Read" [5]

**response fields**: domain, extension, punycode, found, registrar, registrar_id, registrar_name, nameservers (array), registrant contact info, created_date, updated_date, expiration_date, dnssec, status, whois_server [5]

## interface: cloudflare typescript sdk

**package**: `cloudflare` (official sdk) [6]

**version in repo**: 5.2.0

**installation**: `npm install cloudflare` [6]

**initialization**:
```typescript
import Cloudflare from 'cloudflare';

const client = new Cloudflare({
  apiToken: process.env['CLOUDFLARE_API_TOKEN'],
});
```
[6]

**configuration options**:
- `timeout`: request timeout in milliseconds (default: 60 seconds) [6]
- `maxRetries`: automatic retries for failed requests (default: 2) [6]

**zone operations**:
```typescript
// create zone
const zone = await client.zones.create({
  account: { id: '023e105f4ecef8ad9ca31a8372d0c353' },
  name: 'example.com',
  type: 'full',
});

// list zones with pagination
for await (const zone of client.zones.list()) {
  console.log(zone);
}
```
[6]

**requirements**: "TypeScript >= 4.5... Node.js 18 LTS or later" [6]

## domain transfer process

**step-by-step** (per cloudflare docs [4]):

1. unlock domain at current registrar
2. disable DNSSEC (wait 24 hours for TTL expiration)
3. confirm WHOIS privacy is not blocked
4. add domain to cloudflare, update nameservers
5. wait for "Active" status (up to 24 hours)
6. initiate transfer in cloudflare dashboard
7. request auth code from current registrar
8. input auth code in cloudflare
9. provide registrant contact info
10. approve FOA email from cloudflare

**transfer constraints**:
- "Some authorization codes expire within 5-14 days" [4]
- "Each transfer adds one year to your domain registration" [4]
- domain cannot be within 15 days of expiration [4]
- cannot be transferred if registered/transferred within 60 days [4]

## best practices

### within this repo

- use `domain-objects` for domain entities with `static unique` and `static primary` declarations
- use `RefByUnique` for devtime references, `refByUnique()` for runtime references
- follow declastruct dao pattern: `getAll`, `getOne`, `set({ findsert | upsert })`, `del`
- throw `HelpfulError` with treestruct guidance for manual steps

### industry wide

- use bearer token authentication, not global api key [1]
- handle pagination for list operations [6]
- respect TTL constraints per plan tier [2]
- implement idempotent operations via findsert/upsert semantics
