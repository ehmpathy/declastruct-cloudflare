### pattern.1: domain entity definition

**location**: `src/domain.objects/DeclaredCloudflareDomainZone.ts`

**pattern**:
```typescript
export interface DeclaredCloudflareDomainZone {
  id?: string;           // @metadata ŌĆö assigned by cloudflare
  name: string;          // @unique
  type: 'full' | 'partial' | 'secondary';
  status?: 'initializing' | 'pending' | 'active' | 'moved';  // @readonly
}

export class DeclaredCloudflareDomainZone
  extends DomainEntity<DeclaredCloudflareDomainZone>
  implements DeclaredCloudflareDomainZone
{
  public static primary = ['id'] as const;
  public static unique = ['name'] as const;
  public static metadata = ['id'] as const;
  public static readonly = ['status', 'nameServers', 'createdOn'] as const;
}
```

**why**: domain-objects library enables typed entity declarations with identity semantics

**[REUSE]** ŌĆö this pattern is already established and works well for zone, dnsRecord, registration

### pattern.3: set operation (findsert/upsert)

**location**: `src/domain.operations/domainZone/setDomainZone.ts`

**pattern**:
```typescript
export const setDomainZone = async (
  input: PickOne<{
    findsert: DeclaredCloudflareDomainZoneInterface;
    upsert: DeclaredCloudflareDomainZoneInterface;
  }>,
  context: ContextCloudflareApi,
): Promise<HasReadonly<typeof DeclaredCloudflareDomainZone>> => {
  // lookup extant
  const zoneFound = await getOneDomainZone({ by: { unique } }, context);

  // findsert: return if extant
  if (zoneFound && input.findsert) return zoneFound;

  // upsert: update if extant
  if (zoneFound && input.upsert) {
    const updated = await client.zones.edit({ ... });
    return castIntoDeclaredCloudflareDomainZone(updated);
  }

  // create new
  const created = await client.zones.create({ ... });
  return castIntoDeclaredCloudflareDomainZone(created);
};
```

**why**: PickOne ensures exactly one mode; findsert protects extant, upsert overwrites

**[REUSE]** ŌĆö idempotent operations pattern established for zone, dnsRecord, registration

### pattern.5: provider factory

**location**: `src/domain.operations/provider/getDeclastructCloudflareProvider.ts`

**pattern**:
```typescript
export const getDeclastructCloudflareProvider = async (
  input: { apiToken?: string; accountId?: string },
  context: ContextLogTrail,
): Promise<DeclastructCloudflareProvider> => {
  const credentials = getCredentials(input);
  const client = new Cloudflare({ apiToken: credentials.apiToken });

  const daos = {
    DeclaredCloudflareDomainZone: DeclaredCloudflareDomainZoneDao,
    DeclaredCloudflareDomainDnsRecord: DeclaredCloudflareDomainDnsRecordDao,
    DeclaredCloudflareDomainRegistration: DeclaredCloudflareDomainRegistrationDao,
  };

  return new DeclastructProvider({
    name: 'cloudflare',
    daos,
    context: providerContext,
  });
};
```

**why**: factory creates provider with all daos and context for declastruct framework

**[REUSE]** ŌĆö provider already assembles all three entity daos
