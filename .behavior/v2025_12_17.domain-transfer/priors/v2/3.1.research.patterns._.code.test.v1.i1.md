# research.patterns: declastruct-cloudflare test code

## test patterns identified

### pattern.1: integration test with given/when/then

**location**: `src/domain.operations/domainZone/domainZone.integration.test.ts`

**pattern**:
```typescript
import { given, then, useBeforeAll, when } from 'test-fns';

describe('domainZone', () => {
  given('a cloudflare account', () => {
    when('[t0] getAllDomainZones is called', () => {
      then('it should return an array of zones', async () => {
        const zones = await getAllDomainZones(getContext());
        expect(Array.isArray(zones)).toBe(true);
      });
    });
  });
});
```

**why**: BDD-style tests with given/when/then for clear intent

**[REUSE]** — established pattern for all domain operation integration tests

### pattern.2: useBeforeAll for shared setup

**location**: `src/domain.operations/domainZone/domainZone.integration.test.ts`

**pattern**:
```typescript
given('a cloudflare account with at least one zone', () => {
  const zonesData = useBeforeAll(async () => {
    const allZones = await getAllDomainZones(getContext());
    return { zones: allZones, firstZone: allZones[0] ?? null };
  });

  when('[t0] getOneDomainZone is called by primary key', () => {
    then('it should return the zone if zones exist', async () => {
      if (!zonesData.firstZone) return;
      // ... test with zonesData.firstZone
    });
  });
});
```

**why**: `useBeforeAll` runs once for suite, shares result across `then` blocks

**[REUSE]** — efficient setup for tests that share data

### pattern.4: sample context factory

**location**: `src/.test/getSampleCloudflareApiContext.ts`

**pattern**:
```typescript
export const getSampleCloudflareApiContext = (): ContextCloudflareApi &
  ContextLogTrail => {
  const apiToken = process.env.CLOUDFLARE_API_TOKEN;
  const accountId = process.env.CLOUDFLARE_ACCOUNT_ID;

  if (!apiToken) BadRequestError.throw('...');
  if (!accountId) BadRequestError.throw('...');

  return {
    cloudflare: { client: new Cloudflare({ apiToken }), accountId },
    log: { debug: () => {}, info: () => {}, warn: () => {}, error: () => {} },
  };
};
```

**why**: centralized context creation with fail-fast credential check

**[REUSE]** — all integration tests use this factory

### pattern.5: acceptance test via CLI

**location**: `src/contract/sdks/declastruct.acceptance.test.ts`

**pattern**:
```typescript
describe('declastruct CLI workflow', () => {
  given('a declastruct resources file with cloudflare provider', () => {
    when('[t0] a plan is generated via declastruct CLI', () => {
      const prep = useBeforeAll(async () => {
        execSync(
          `npx declastruct plan --wish ${resourcesFile} --into ${planFile}`,
          { stdio: 'inherit', env: process.env, cwd: repoRoot },
        );
        return { plan: JSON.parse(readFileSync(planFile, 'utf-8')) };
      });

      then('creates a valid plan file', () => {
        expect(existsSync(planFile)).toBe(true);
      });
    });
  });
});
```

**why**: black-box tests via declastruct CLI commands

**[REUSE]** — validates end-to-end workflow

### pattern.7: read-only integration tests

**location**: `src/domain.operations/domainZone/domainZone.integration.test.ts`

**pattern**: integration tests only call read operations (getAll, getOne)
- avoids resource creation in demo account
- safe to run repeatedly
- tests creation via unit tests with mocked responses

**why**: demo account may have constraints; read-only is safe

**[REUSE]** — safe for CI without resource cleanup

## test infra patterns

### pattern.9: test directory structure

**pattern**:
```
src/
├── .test/
│   ├── getSampleCloudflareApiContext.ts
│   └── getMockedCloudflareApiContext.ts
├── domain.objects/
│   ├── DeclaredCloudflareDomainZone.ts
│   └── DeclaredCloudflareDomainZone.test.ts    # unit test collocated
├── domain.operations/
│   └── domainZone/
│       ├── setDomainZone.ts
│       ├── setDomainZone.test.ts               # unit test collocated
│       └── domainZone.integration.test.ts      # integration test
└── contract/sdks/
    ├── index.ts
    ├── declastruct.acceptance.test.ts          # acceptance test
    └── .test/assets/resources.acceptance.ts
```

**why**: collocated tests for locality; shared test utils in `.test/`

**[REUSE]** — established directory structure

### pattern.10: jest config separation

**location**: project root
- `jest.unit.config.ts` — unit tests (`.test.ts`)
- `jest.integration.config.ts` — integration tests (`.integration.test.ts`)
- `jest.acceptance.config.ts` — acceptance tests (`.acceptance.test.ts`)

**why**: separate configs for different test tiers; different timeouts, env vars

**[REUSE]** — standard test tier separation
