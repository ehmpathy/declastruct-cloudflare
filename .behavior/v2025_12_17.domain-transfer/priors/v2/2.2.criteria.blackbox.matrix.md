# criteria.blackbox.matrix: declastruct-cloudflare domain management

## matrix.1 = zone management

| ind: operation | ind: zone extant? | ind: has records? | dep: outcome | dep: idempotent? |
|----------------|-------------------|-------------------|--------------|------------------|
| list | - | - | returns all zones | yes |
| create | no | - | creates zone | yes |
| create | yes | - | returns unchanged | yes |
| delete | yes | no | removes zone | yes |
| delete | yes | yes | deletes records first, then zone | yes |
| delete | no | - | no-op (already absent) | yes |

**gaps**: none. all combinations covered.

## matrix.2 = dns record management

| ind: operation | ind: record extant? | ind: values match? | ind: mode | ind: zone extant? | dep: outcome | dep: idempotent? |
|----------------|---------------------|--------------------|-----------|--------------------|--------------|------------------|
| list | - | - | - | yes | returns all records | yes |
| create | no | - | findsert | yes | creates record | yes |
| create | no | - | upsert | yes | creates record | yes |
| create | yes | yes | findsert | yes | returns unchanged | yes |
| create | yes | yes | upsert | yes | returns unchanged | yes |
| create | yes | no | findsert | yes | throws conflict error | yes |
| create | yes | no | upsert | yes | updates record | yes |
| delete | yes | - | - | yes | removes record | yes |
| delete | no | - | - | yes | no-op (already absent) | yes |
| create | no | - | - | no | creates zone first, then record | yes |

**gaps**:
- list operation when zone does not exist — not specified (likely throws error)
- delete operation when zone does not exist — not specified

## summary of gaps

| matrix | gap | severity | action |
|--------|-----|----------|--------|
| dns record | list when zone absent | low | add criterion or rely on cloudflare error |
| dns record | delete when zone absent | low | add criterion or rely on cloudflare error |
| error cases | network failure | medium | add criterion for transient error behavior |
| error cases | rate limit | medium | add criterion for rate limit behavior |

**recommendation**: the low-severity gaps can be addressed via cloudflare's native error responses. the medium-severity gaps (network/rate limit) are worth adding criteria for, but can be deferred to implementation phase if needed.
