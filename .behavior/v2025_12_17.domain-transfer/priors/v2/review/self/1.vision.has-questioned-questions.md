## open questions from prior reviews

### question 1: does cloudflare api support zone operations?

**source**: devil's advocate review
**can answer via research**: YES

**answer**: YES. cloudflare zones api supports:
- `GET /zones` — list zones
- `POST /zones` — create zone
- `GET /zones/{zone_id}` — get zone details
- `PATCH /zones/{zone_id}` — update zone
- `DELETE /zones/{zone_id}` — delete zone

findsert/upsert semantics achievable via get-then-create/update.

**source**: [Cloudflare API | Zones](https://developers.cloudflare.com/api/resources/zones/)

---

### question 2: does cloudflare api support dns record operations?

**source**: devil's advocate review
**can answer via research**: YES

**answer**: YES. cloudflare dns records api supports:
- `GET /zones/{zone_id}/dns_records` — list records
- `POST /zones/{zone_id}/dns_records` — create record
- `PATCH /zones/{zone_id}/dns_records/{record_id}` — partial update
- `PUT /zones/{zone_id}/dns_records/{record_id}` — overwrite
- `DELETE /zones/{zone_id}/dns_records/{record_id}` — delete

findsert/upsert semantics achievable.

**source**: [Cloudflare API | DNS › Records](https://developers.cloudflare.com/api/resources/dns/subresources/records/methods/create/)

### question 4: is dns record unique key `(zone_id, name, type)`?

**source**: devil's advocate review
**can answer via research**: YES

**answer**: PARTIALLY. dns records are identified by:
- `zone_id` (required)
- `id` (record id, assigned by cloudflare)

to find by `(name, type)`, use list endpoint with filters then match.

note: same name can have multiple records of same type (e.g., multiple A records for round-robin). true unique key is `(zone_id, id)`.

**implication**: findsert must decide how to handle multiple records with same (name, type). blueprint decision.

### question 6: can cloudflare api register or transfer domains?

**source**: 5whys review
**can answer via research**: YES

**answer**: NO. cloudflare api supports:
- `GET /registrar/domains` — list registered domains
- `PUT /registrar/domains/{domain}` — update options (auto_renew, locked, privacy)

but NO endpoints for:
- register new domain
- initiate domain transfer

**implication**: vision is correct that registration/transfer are "guided" not automated.

**source**: [Cloudflare API | Registrar › Domains › Update Domain](https://developers.cloudflare.com/api/resources/registrar/subresources/domains/methods/update/)
