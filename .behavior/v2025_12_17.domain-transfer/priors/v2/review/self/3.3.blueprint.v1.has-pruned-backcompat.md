## backcompat.3: setDomainRegistration signature

**what changes**: adds BadRequestError throw when domain not found.

**is this a break?**: no — prior code threw UnexpectedCodePathError. new code throws BadRequestError. both are errors. callers that catch errors will still work.

**backcompat concern**: callers that specifically catch UnexpectedCodePathError will miss the new error.

**assessment**: unlikely any caller catches by error type. this is a fix, not a break.

**verdict**: NOT A BACKCOMPAT CONCERN — error type change is a bugfix.

## backcompat.4: findsert attribute diff check

**what changes**: findsert now throws BadRequestError if found object has different attributes.

**is this a break?**: potentially — callers that relied on findsert to silently return mismatched objects will now see errors.

**backcompat concern**: this changes behavior from "silent mismatch" to "explicit error".

**assessment**: this is a FIX. silent mismatch is a bug per rule.require.idempotency. callers that relied on bug behavior should not be preserved.

**verdict**: NOT A BACKCOMPAT CONCERN — bug fix takes precedence.
