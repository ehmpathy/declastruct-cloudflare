## deeper questions first

### do we need the entire WHOIS feature?

**question**: could we skip WHOIS entirely?

**analysis**:
- setDomainRegistration needs to detect: transfer vs purchase
- without WHOIS, we'd throw generic "not found, transfer or purchase manually"
- with WHOIS, we give specific guidance: "transfer from godaddy" or "available to purchase"
- the vision explicitly calls for this distinction

**verdict**: KEEP — the detection adds user value

---

### do we need DeclaredCloudflareDomainWhoisRecord as a separate file?

**question**: does the domain literal need its own file in domain.objects/?

**analysis**:
- it's a declared record with a unique key (domain)
- follows the same pattern as other DeclaredCloudflare* domain objects
- separate file maintains consistency with codebase patterns
- unit test validates construction and schema

**verdict**: KEEP — consistent with codebase patterns for declared domain objects

---

### do we need the `domain` field in WhoisRecord?

**question**: DeclaredCloudflareDomainWhoisRecord has `domain: string` — is this needed?

**analysis**:
- domain is the unique key for this record
- required for identity — you can't identify a WHOIS record without knowing which domain it's for
- follows the same pattern as other domain objects with unique keys

**verdict**: KEEP — required for unique key identity

## components questioned

### DeclaredCloudflareDomainWhoisRecord fields

**question**: do we need all 7 fields?

**analysis**: the guidance logic in setDomainRegistration only uses:
- `found` — to check if domain is registered
- `registrar` — to check if it has a registrar
- `registrarName ?? registrar` — to display the registrar name

these fields are NOT used in guidance:
- `nameservers` — not used
- `expirationDate` — not used
- `createdDate` — not used

**verdict**: DELETE unused fields. simplify DeclaredCloudflareDomainWhoisRecord to:
```
DeclaredCloudflareDomainWhoisRecord
├── domain: string
├── found: boolean
├── registrar: string | null
└── registrarName: string | null
```
