# blueprint: declastruct-cloudflare domain management

## overview

this blueprint declares the implementation plan for declarative control of cloudflare domains via declastruct.

**critical**: do NOT trust extant implementation. all retained files marked `[○]` require full re-review at execution time to ensure they match patterns and criteria. the prior implementation may have defects.

### focus areas

1. re-review all extant zone/dnsRecord/registration operations for correctness
2. add WHOIS lookup for transfer/purchase detection
3. extend setDomainRegistration with HelpfulError guidance
4. ensure all operations have proper tests

## setDomainRegistration guidance logic

```typescript
// current: throws UnexpectedCodePathError when not found
// update: throw BadRequestError with guidance

if (!regFound) {
  // lookup via WHOIS to determine transfer vs purchase
  const domainWhoisRecordFound = await getOneDomainWhoisRecord({ domain: regDesired.name }, context);

  if (domainWhoisRecordFound?.found && domainWhoisRecordFound.registrar) {
    // domain registered elsewhere → transfer guidance
    BadRequestError.throw(
      `domain "${regDesired.name}" not found in cloudflare registrar

⚠️ transfer required
   │
   ├─ at ${domainWhoisRecordFound.registrarName ?? domainWhoisRecordFound.registrar} (current registrar)
   │  └─ steps
   │     ├─ 1. disable dnssec
   │     ├─ 2. disable whois privacy
   │     ├─ 3. unlock domain
   │     └─ 4. get auth code
   │
   ├─ at cloudflare
   │  └─ steps
   │     ├─ 5. enter domain + auth code at https://dash.cloudflare.com/${accountId}/domains/transfer
   │     ├─ 6. confirm contact
   │     └─ 7. complete transfer
   │
   └─ post-initiation
      ├─ 8. approve FOA email
      ├─ 9. wait up to 5 days
      └─ 10. re-run declastruct apply`,
      { domain: regDesired.name, registrar: domainWhoisRecordFound.registrar, accountId },
    );
  } else {
    // domain not registered → purchase guidance
    BadRequestError.throw(
      `domain "${regDesired.name}" not found in cloudflare registrar

⚠️ purchase required
   │
   ├─ domain is available for registration
   │
   └─ at cloudflare
      └─ steps
         ├─ 1. search for "${regDesired.name}" at https://dash.cloudflare.com/${accountId}/domains/register
         ├─ 2. add to cart
         ├─ 3. complete purchase
         └─ 4. re-run declastruct apply`,
      { domain: regDesired.name, accountId },
    );
  }
}
```

### defect.2: findsert returns without diff check

**files**:
- `domain.operations/domainZone/setDomainZone.ts:52`
- `domain.operations/domainDnsRecord/setDomainDnsRecord.ts:64`
- `domain.operations/domainRegistration/setDomainRegistration.ts:62`

**fix** (example for setDomainZone):
```typescript
// before:
if (input.findsert) return zoneFound;

// after:
if (input.findsert) {
  // check for attribute diff
  const hasTypeDiff = zoneDesired.type !== zoneFound.type;
  const hasPausedDiff = zoneDesired.paused !== undefined && zoneDesired.paused !== zoneFound.paused;

  if (hasTypeDiff || hasPausedDiff)
    BadRequestError.throw(
      'cannot findsert zone; zone exists with different attributes',
      { zoneDesired, zoneFound, diffs: { type: hasTypeDiff, paused: hasPausedDiff } },
    );

  return zoneFound;
}
```

**why**: per rule.require.idempotency, findsert must throw if attributes differ

### defect.6: `as` casts in cast functions

- `domain.operations/domainDnsRecord/castIntoDeclaredCloudflareDomainDnsRecord.ts:39,46-48,57-58`

**fix approach**:
1. for enum-like fields (type, status): validate against allowed values, throw if invalid
2. for SDK response shape: define explicit interface matching SDK types

```typescript
// example for zone type:
// before:
type: input.type as 'full' | 'partial' | 'secondary',

// after:
type: (() => {
  const allowed = ['full', 'partial', 'secondary'] as const;
  if (!allowed.includes(input.type as typeof allowed[number]))
    throw new UnexpectedCodePathError('invalid zone type from API', { type: input.type });
  return input.type as typeof allowed[number];
})(),
```

**why**: per rule.forbid.as-cast, validate at runtime instead of blind cast
