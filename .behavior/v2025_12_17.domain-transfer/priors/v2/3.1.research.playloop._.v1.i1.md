## gap analysis

### gap.1: live api dependency

**current state**: integration tests require live cloudflare api

**impact**:
- tests fail without credentials
- rate limits may affect bulk test runs
- test data persists in demo account

**mitigation in place**:
- credential skip pattern
- read-only integration tests
- demo account with limited resources

### gap.2: acceptance test coverage

**current state**: acceptance tests exist but return empty resources

**impact**:
- plan/apply workflow verified but no actual changes tested

**opportunity**: add test zone when available to verify full workflow

### gap.3: error guidance verification

**current state**: no dedicated tests for HelpfulError guidance content

**impact**:
- transfer/purchase guidance messages not snapshot tested

**opportunity**: add snapshot tests for error message structure

## experience reproduction patterns

### pattern.1: api surface verification

**entry point**: SDK export (`import { getDeclastructCloudflareProvider } from 'declastruct-cloudflare'`)

**user action simulation**: call dao methods with test inputs

**assertion**: verify return type matches domain object

```typescript
given('a cloudflare account', () => {
  when('getAllDomainZones is called', () => {
    then('it should return an array of zones', async () => {
      const zones = await getAllDomainZones(getContext());
      expect(Array.isArray(zones)).toBe(true);
    });
  });
});
```

### pattern.2: error condition verification

**entry point**: set operation with invalid state

**user action simulation**: call setDomainRegistration for unregistered domain

**assertion**: verify HelpfulError with correct guidance

```typescript
given('a domain not in cloudflare registrar', () => {
  when('setDomainRegistration is called', () => {
    then('it should throw BadRequestError with transfer guidance', async () => {
      const error = await getError(
        setDomainRegistration({ findsert: registration }, context)
      );
      expect(error).toBeInstanceOf(BadRequestError);
      expect(error.message).toContain('transfer required');
    });
  });
});
```

### pattern.3: CLI workflow verification

**entry point**: declastruct CLI commands

**user action simulation**: execSync with plan/apply commands

**assertion**: verify output files and return codes

```typescript
given('a declastruct resources file', () => {
  when('a plan is generated via declastruct CLI', () => {
    then('creates a valid plan file', async () => {
      execSync(
        `npx declastruct plan --wish ${resourcesFile} --into ${planFile}`,
        { stdio: 'inherit', env: process.env, cwd: repoRoot },
      );
      expect(existsSync(planFile)).toBe(true);
    });
  });
});
```

## recommended playloop workflow

### for mechanic development

1. **write unit test** for domain object or pure function
2. **run unit tests**: `npm run test:unit -- path/to/file`
3. **implement** the minimal code to pass
4. **run integration test** against live api
5. **verify acceptance test** still passes

### for human verification

1. **source credentials**: `source .agent/repo=.this/role=any/skills/use.demo.cloudflare.creds.sh`
2. **run full test suite**: `npm run test`
3. **review test output** for failures
