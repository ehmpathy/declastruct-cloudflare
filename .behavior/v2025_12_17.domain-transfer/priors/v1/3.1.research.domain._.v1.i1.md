## domain objects

### entities

#### 2. DnsRecord

represents a single dns record within a zone.

**unique key**: `{ zone_id, name, type, content }` (composite - same name can have multiple records)
**primary key**: `id` (cloudflare record identifier)

| field | type | description | source |
|-------|------|-------------|--------|
| `id` | string | record identifier | [7] |
| `zone_id` | string | parent zone identifier | [7] |
| `name` | string | dns record name (punycode) | [7], [8] |
| `type` | enum | record type (see below) | [7] |
| `content` | string | record value | [7], [8] |
| `ttl` | number | time-to-live (60-86400, or 1 for auto) | [7], [8] |
| `proxied` | boolean | cloudflare proxy status | [7], [8] |
| `proxiable` | boolean | whether record can be proxied | [7] |
| `comment` | string | optional annotation | [7], [8] |
| `tags` | string[] | array of name:value tags | [7], [8] |
| `priority` | number | for MX records (lower = preferred) | [14] |
| `data` | object | structured data for complex types (CAA, SRV, etc.) | [14] |
| `settings` | object | record-specific settings | [7], [14] |
| `created_on` | timestamp | creation timestamp | [7] |
| `modified_on` | timestamp | last modification timestamp | [7] |
| `meta` | object | cloudflare metadata | [14] |

**supported dns record types [7]**:
- standard: `A`, `AAAA`, `CNAME`, `MX`, `NS`, `PTR`, `TXT`
- advanced: `CAA`, `CERT`, `DNSKEY`, `DS`, `HTTPS`, `LOC`, `NAPTR`, `OPENPGPKEY`, `SMIMEA`, `SRV`, `SSHFP`, `SVCB`, `TLSA`, `URI`

**quote [8]**: "Common Fields (all record types): name (string), type (string), ttl (number | 1), comment (string, optional), proxied (boolean, optional), tags (array, optional), settings (object, optional)"

**quote [7]**: "A/AAAA records cannot coexist with CNAME records on the same name. NS records cannot coexist with other record types on the same name."

### domain transfer operations

transfers are primarily a manual/ui process with api support for status checking.

**transfer requirements [11]**:
1. domain must use cloudflare for authoritative dns (full setup)
2. dnssec must be disabled at current registrar
3. domain must not be within 60 days of registration or previous transfer
4. domain must not have locks like `clientTransferProhibited`
5. valid payment method required

**quote [11]**: "All domains using Cloudflare Registrar must use Cloudflare for authoritative DNS on a full setup."

**quote [11]**: "Cannot transfer within 60 days of registration or previous transfer"

**transfer statuses [11]**:
- `Transfer in progress` - awaiting previous registrar confirmation
- `Pending approval` - registrar reviewing request (5-day window)
- `Transfer rejected` - request denied

**quote [11]**: "Transfer in progress: Awaiting previous registrar confirmation (typically under 24 hours)"

## authentication

**api token (recommended) [4]**:
```
Authorization: Bearer $CLOUDFLARE_API_TOKEN
```

**required permissions**:
- zones: "Zone Zone Edit" or "Zone DNS Edit" [10]
- dns records: "DNS Read" or "DNS Write" [7]

## key constraints

1. **zone type [10]**: must be "full" for registrar features
2. **dns record conflicts [8]**: A/AAAA cannot coexist with CNAME on same name
3. **ns record conflicts [8]**: NS cannot coexist with other types on same name
4. **ttl range [8]**: 60-86400 seconds (30+ for enterprise), or 1 for automatic
5. **domain names [8]**: represented in punycode format
