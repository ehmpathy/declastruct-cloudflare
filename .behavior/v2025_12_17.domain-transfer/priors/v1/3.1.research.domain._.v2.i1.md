## declastruct guidance procedures

for non-automatable SETs, we guide users through completion via HelpfulErrors.

### procedure: domain registration

```
when: user declares a RegistrarDomain that doesn't exist
action: throw HelpfulError with guidance

HelpfulError {
  message: "domain registration requires manual action",
  guidance: [
    "1. navigate to cloudflare dashboard → registrar → register domains",
    "2. search for domain: ${domainName}",
    "3. select purchase and complete checkout",
    "4. verify domain appears in account",
    "5. re-run declastruct apply"
  ],
  link: "https://dash.cloudflare.com/?to=/:account/domains/register"
}
```

### procedure: domain transfer initiate

```
when: user declares a RegistrarDomain with transfer_in intent
action: throw HelpfulError with guidance

HelpfulError {
  message: "domain transfer requires manual steps at both registrars",
  guidance: [
    "at current registrar:",
    "  1. log into current registrar account",
    "  2. disable dnssec (note ds record ttl, wait for propagation)",
    "  3. disable whois privacy",
    "  4. unlock domain (remove clientTransferProhibited)",
    "  5. request authorization code",
    "",
    "at cloudflare:",
    "  6. navigate to dashboard → transfer domains",
    "  7. enter domain name and auth code",
    "  8. confirm contact information",
    "  9. complete transfer initiation",
    "",
    "post-initiation:",
    "  10. approve form of authorization (foa) email",
    "  11. wait for transfer completion (up to 5 days)",
    "  12. re-run declastruct apply"
  ],
  link: "https://dash.cloudflare.com/?to=/:account/domains/transfer"
}
```

**quote [18]**: "Unlock the domain (remove registrar lock/'clientTransferProhibited' status)... Disable WHOIS privacy if enabled... Obtain authorization code from current registrar"

### procedure: zone activation (pending nameservers)

```
when: Zone.status === "pending"
action: throw HelpfulError with guidance

HelpfulError {
  message: "zone pending activation - nameserver change required",
  guidance: [
    "1. log into your domain registrar (${zone.original_registrar})",
    "2. update nameservers to cloudflare's assigned servers:",
    zone.name_servers.map(ns => `     - ${ns}`).join('\n'),
    "3. wait for dns propagation (up to 24-48 hours)",
    "4. zone will auto-activate once verified",
    "5. re-run declastruct apply to continue"
  ],
  currentNameservers: zone.original_nameservers,
  requiredNameservers: zone.name_servers
}
```

## domain object relationships

### entity hierarchy (treestruct)

```
Account
└── Zone (1:many)
    ├── DnsRecord (1:many)
    │   └── DnsRecordSettings (1:1, embedded)
    └── RegistrarDomain (1:1, optional)
        ├── RegistrantContact (1:1, embedded)
        └── TransferIn (1:1, embedded)
```

### relationship types

| parent | child | cardinality | dependency | notes |
|--------|-------|-------------|------------|-------|
| Account | Zone | 1:many | required | zones belong to accounts |
| Zone | DnsRecord | 1:many | zone must exist | records are zone-scoped |
| Zone | RegistrarDomain | 1:1 | zone must exist & be "full" type | registrar requires cloudflare dns |
| DnsRecord | DnsRecordSettings | 1:1 | embedded | optional settings per record |
| RegistrarDomain | RegistrantContact | 1:1 | embedded | contact info |
| RegistrarDomain | TransferIn | 1:1 | embedded | transfer status |

**quote [11]**: "All domains using Cloudflare Registrar must use Cloudflare for authoritative DNS on a full setup."

### dependency chain for wish fulfillment

```
1. Zone (must exist and be active)
   ↓ requires
2. Nameserver delegation (external action)
   ↓ enables
3. DnsRecord management (fully automatable)
   ↓ parallel
4. RegistrarDomain transfer/registration (manual + api)
   ↓ enables
5. RegistrarDomain settings (auto_renew, privacy, locked)
```

## domain objects (extended)

### entities

#### 1. Zone

- **unique key**: `name` (domain name)
- **primary key**: `id`
- **automatable**: ✅ full crud via api
- **plan restrictions**: type="partial"|"secondary" requires business/enterprise

#### 2. DnsRecord

- **unique key**: `{ zone_id, name, type, content }` (composite)
- **primary key**: `id`
- **automatable**: ✅ full crud via api
- **plan restrictions**: ttl minimum 60s (30s for enterprise)

#### 3. RegistrarDomain

- **unique key**: `domain_name`
- **primary key**: `id`
- **automatable**: ⚠️ partial (read + update settings only)
- **not automatable**: registration, transfer initiate
- **plan restrictions**: none (available all plans)

### literals (value objects)

#### 1. RegistrantContact

embedded in RegistrarDomain. immutable except via dashboard.

#### 2. TransferIn

transfer status indicators. read-only via api.

| field | meaning | source |
|-------|---------|--------|
| `accept_foa` | needs foa acceptance | [4] |
| `approve_transfer` | needs transfer approval | [4] |
| `can_cancel_transfer` | transfer is cancellable | [4] |
| `disable_privacy` | needs privacy disabled | [4] |
| `enter_auth_code` | needs auth code entry | [4] |
| `unlock_domain` | needs domain unlock | [4] |

#### 3. DnsRecordSettings

embedded in DnsRecord. type-specific config.

#### 4. ZoneMeta

embedded in Zone. contains quotas, cdn/dns mode flags.

### events

no explicit domain events. track changes via:
- `created_on` timestamps
- `modified_on` timestamps
- Zone.status changes (initializing → pending → active)
- TransferIn status field changes

## sdk method summary

### fully automatable (sdk available)

```typescript
// zones
client.zones.create({ account: { id }, name, type })
client.zones.list({ ...params })
client.zones.get({ zone_id })
client.zones.update(zoneId, { ...params })
client.zones.delete({ zone_id })

// dns records
client.dns.records.create({ zone_id, name, type, content, ttl, proxied })
client.dns.records.list({ zone_id })
client.dns.records.get(recordId, { zone_id })
client.dns.records.update(recordId, { zone_id, ...params })
client.dns.records.edit(recordId, { zone_id, ...params })
client.dns.records.delete(recordId, { zone_id })
client.dns.records.batch({ zone_id, deletes, patches, puts, posts })
client.dns.records.import({ zone_id, file })
client.dns.records.export({ zone_id })

client.registrar.domains.list({ account_id })
client.registrar.domains.get(domainName, { account_id })
client.registrar.domains.update(domainName, { account_id, auto_renew, locked, privacy })
```

## key constraints for declastruct implementation

1. **zone type must be "full"** for registrar features to work [11]
2. **zone must be "active"** before dns records can be managed
3. **nameserver delegation is external** - cannot be automated
4. **domain registration/transfer requires dashboard** - guide via HelpfulError
5. **ttl minimum varies by plan** - validate 60s (free) vs 30s (enterprise) [21]
6. **dns record conflicts** - A/AAAA cannot coexist with CNAME on same name [8]
7. **ns record conflicts** - NS cannot coexist with other types on same name [8]
8. **transfer requires 60-day wait** after registration or previous transfer [11]
