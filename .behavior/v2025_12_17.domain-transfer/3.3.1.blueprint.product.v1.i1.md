## fixes required

### fix.1: DnsRecord unique key

add `content` to unique key to support multiple records with same (name, type) for load distribution.

- current: `['zone', 'name', 'type']`
- required: `['zone', 'name', 'type', 'content']`
- files: `DeclaredCloudflareDomainDnsRecord.ts`, `getOneDomainDnsRecord.ts`, `setDomainDnsRecord.ts`

### fix.2: error type for registration guidance

change error type from UnexpectedCodePathError to BadRequestError for transfer/purchase guidance.

- current: `throw new UnexpectedCodePathError(...)`
- required: `BadRequestError.throw(...)`
- why: UnexpectedCodePathError signals internal bug; BadRequestError signals user-actionable issue
- files: `setDomainRegistration.ts`

### fix.3: findsert diff check

add attribute diff check to findsert operations — if extant record has different attributes, throw BadRequestError instead of silent return of stale data.

- current: findsert returns extant without attribute comparison
- required: findsert compares attributes, throws if diff detected
- files: `setDomainZone.ts`, `setDomainDnsRecord.ts`, `setDomainRegistration.ts`

### fix.4: forbidden term rename

rename function to avoid forbidden term per rule.forbid.term.

- current: uses forbidden term in function name
- required: `expandZoneRef`
- files: `domainDnsRecord/expandZoneRef.ts`, `domainDnsRecord/expandZoneRef.test.ts`

## execution approach

### phase 0: verify factory (unit tests)

1. run `npm run test:unit`
2. verify all unit tests pass
3. if failures: review and fix affected code

### phase 1: review domain objects

1. review each domain object against criteria
2. verify static unique, primary, readonly declarations
3. verify interface matches cloudflare api shape
4. verify nested declarations for refs

### phase 2: review domain operations

1. review each operation against declastruct patterns
2. verify findsert/upsert/del semantics
3. verify error behavior (BadRequestError for guidance)
4. verify cast functions match api response shapes

### phase 3: review daos

1. verify genDeclastructDao usage
2. verify dao methods delegate to operations
3. verify provider assembly

### phase 4: verify integration (requires credentials)

1. human provisions credentials via keyrack.fill.sh
2. run `npm run test:integration`
3. verify all integration tests pass

### phase 5: verify acceptance (requires credentials)

1. run `npm run test:acceptance`
2. verify declastruct plan/apply workflow works
