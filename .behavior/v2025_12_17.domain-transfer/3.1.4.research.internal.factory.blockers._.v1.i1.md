## access and credentials

### status

**blocker** — integration tests fail without credentials.

### evidence

```
FAIL src/domain.operations/domainZone/domainZone.integration.test.ts
● Test suite failed to run

  credentials not found. setup via:
    1. npx rhachet roles init --repo ehmpathy --role mechanic --init keyrack.ehmpath
    2. npx rhx keyrack set --key CLOUDFLARE_ACCOUNT_ID --env test
    3. npx rhx keyrack set --key CLOUDFLARE_API_TOKEN --env test
```

### required credentials

| key | purpose | who provisions |
|-----|---------|----------------|
| CLOUDFLARE_ACCOUNT_ID | account scope for API calls | human |
| CLOUDFLARE_API_TOKEN | authentication | human |

### provision steps

```sh
# 1. init ehmpath keyrack (one-time per machine)
npx rhachet roles init --repo ehmpathy --role mechanic --init keyrack.ehmpath

# 2. unlock keyrack
npx rhx keyrack unlock --owner ehmpath --prikey ~/.ssh/ehmpath --env all

# 3. fill credentials
./.agent/repo=.this/role=any/skills/keyrack.fill.sh --env test
```

### what's blocked without credentials

| capability | blocked? |
|------------|----------|
| unit tests | no |
| integration tests | **yes** |
| acceptance tests | **yes** |
| build | no |
| type check | no |

### resolution

human must provision credentials. mechanic cannot proceed with integration/acceptance verification until credentials are available.
