## claim.1: cloudflare registrar pricing model

### citations

[1] https://www.cloudflare.com/learning/dns/what-is-cloudflare-registrar/

> "Cloudflare Registrar is a service for registering domain names that doesn't mark up the price."

[2] https://yourtechcompass.com/cloudflare-registrar-review/

> "offering essentially the same services as other domain name registrars but without markups, without increased renewal fees"

### label

**[FACT]**: cloudflare registrar does not mark up wholesale domain prices

### relevance

users expect cost transparency. declastruct-cloudflare guidance for domain purchase/transfer should acknowledge cloudflare's at-cost pricing as a value proposition.

## claim.2: ICANN 60-day transfer restriction

### citations

[3] https://developers.cloudflare.com/registrar/get-started/transfer-domain-to-cloudflare/

> "Your domain must have been registered at least 60 days ago and not transferred in the last 60 days."

[4] https://community.cloudflare.com/t/icann-60-day-rule/147474

> "The rule also prohibits transfers if the WHOIS Registrant contact information was modified in the last 60 days (even if redacted)."

### label

**[FACT]**: ICANN prohibits domain transfers within 60 days of registration, previous transfer, or WHOIS registrant change

### relevance

transfer guidance must warn users about 60-day restrictions. HelpfulError should detect and explain this blocker.

## claim.4: DNSSEC blocks domain transfer

### citations

[6] https://developers.cloudflare.com/registrar/troubleshooting/

> "Active DNSSEC at your current registrar will block the transfer."

> "You must disable DNSSEC and wait for the DS record TTL to expire (usually 24 hours) before retrying."

### label

**[FACT]**: active DNSSEC blocks domain transfer; requires disable + 24h wait

### relevance

transfer guidance must include DNSSEC disable step with TTL wait warning.

## claim.12: idempotence ensures consistent results

### citations

[15] https://www.splunk.com/en_us/blog/learn/idempotent-design.html

> "Idempotence refers to an operation that can be applied multiple times without changing the outcome beyond the initial application."

> "Ensuring consistent results regardless of how many times a request is repeated."

### label

**[FACT]**: idempotent operations produce same result regardless of how many times executed

### relevance

## claim.16: what if terraform is not suitable?

### citations

[20] https://aztoso.com/misc/provision-private-dns-zone/

> "If you can't use ARM templates or Terraform, but you still want your deployments to be idempotent, you have an option to create idempotent deployments using scripts."

### label

**[KHUE]**: when is a custom solution preferable to terraform for DNS management?

### relevance

declastruct-cloudflare is a custom solution. we should consider: why not just use terraform cloudflare provider?

**answer candidates**:
- typescript-native integration with extant declastruct ecosystem
- unified pattern across aws, github, cloudflare
- custom guidance for operations terraform cannot automate (transfer, purchase)

## claim.17: account ownership best practices

### citations

[21] https://developers.cloudflare.com/fundamentals/reference/best-practices/

> "Ensure the registrant of the domain name is your organization's name."

> "Use an email address that is not tied to the domain name itself."

### label

**[FACT]**: domain registrant should be organization name; email should not depend on the domain

### relevance

good practice to document. if domain DNS fails, email tied to that domain becomes unreachable for recovery.

### key insights

1. **ICANN rules are strict**: 60-day restrictions, auth code expiration, DNSSEC blocks — transfer guidance must be comprehensive

2. **infrastructure as code is validated**: multiple sources confirm the value of declarative DNS management

3. **idempotency is essential**: prevents duplicates, enables safe retries, produces consistent results

4. **terraform alternative is valid use case**: when custom guidance and ecosystem integration are required
