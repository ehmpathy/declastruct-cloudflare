## cross-resource field naming

|---------|------|-----------|--------------|-------------|------------|

| domain name | name | name | name | domain | yes but justified |

**"domain" vs "name" divergence analysis**:

the WHOIS API parameter is `domain`, not `name`. this is not a convention violation — it's API alignment:

```typescript
// cloudflare SDK (extant, not our code)
client.intel.whois.get({ account_id, domain })
                                      ^^^^^^ API uses "domain"
```

zones and registrations use `name` because they're cloudflare-managed resources. WHOIS uses `domain` because it's a lookup parameter for external domains. different semantic contexts, appropriate term choices.
