## adherence verification

### domain object adherence

**DeclaredCloudflareDomainZone**:
- vision says: unique=[name], primary=[id], type='full'|'partial'
- blueprint says: unique key [name], primary key [id]
- **adherent**: blueprint matches vision exactly. no deviation.

**DeclaredCloudflareDomainDnsRecord**:
- vision says: zone ref, name, type, content, ttl, proxied, priority (for MX)
- blueprint says: operations for getAll, getOne, set, del with zone reference
- **adherent**: blueprint matches vision. fix for unique key is tracked (content must be included).

**DeclaredCloudflareDomainRegistration**:
- vision says: autoRenew, locked, privacyProtection
- blueprint says: operations for getAll, getOne, set (no del - intentional)
- **adherent**: blueprint matches vision. no del is correct per cloudflare api limits.

**Nested objects (WhoisRecord, DnsRecordSettings, RegistrantContact, TransferIn)**:
- vision says: whois for detection, transferIn tracks progress
- blueprint says: all nested objects in filediff
- **adherent**: blueprint includes all nested objects per vision.

### operation adherence

**zone operations**:
- criteria says: findsert creates, findsert returns extant, upsert updates, del removes, del is idempotent
- blueprint codepath tree shows:
  - if not found: create via api
  - if found + findsert: return extant
  - if found + upsert: update via api
- **adherent**: codepath matches criteria semantics exactly.

**dns record operations**:
- criteria says: same findsert/upsert/del semantics, plus support for multiple A records
- blueprint codepath tree shows:
  - expand zone ref to zone id (prerequisite step)
  - same findsert/upsert/del branches as zone
- **adherent**: blueprint matches criteria. unique key fix tracked for multiple A records.

**registration operations**:
- criteria says: upsert updates settings, findsert is idempotent
- blueprint codepath tree shows:
  - if not found: query whois → throw guidance
  - if found + findsert: check diff, return extant
  - if found + upsert: update via api
- **adherent**: blueprint matches criteria. guidance flow added when domain not in cloudflare.

### guidance adherence

**transfer guidance**:
- vision shows exact treestruct format with registrar steps:
  ```
  ⚠️ transfer required
     ├─ at godaddy (current registrar)
     │  ├─ steps: disable dnssec, disable whois privacy, unlock domain, get auth code
     ├─ at cloudflare
     │  ├─ steps: enter domain + auth code, confirm contact, complete transfer
     └─ post-initiation
        ├─ approve FOA email, wait up to 5 days, re-run apply
  ```
- blueprint codepath shows: "if whois.found: throw transfer guidance"
- **adherent**: blueprint flow produces the guidance format from vision.

**purchase guidance**:
- vision shows treestruct format:
  ```
  ⚠️ purchase required
     └─ at cloudflare: search, add to cart, complete purchase, re-run apply
  ```
- blueprint codepath shows: "if !whois.found: throw purchase guidance"
- **adherent**: blueprint flow matches vision's purchase guidance format.

**whois detection**:
- vision says: "uses cloudflare's WHOIS API (`/accounts/{account_id}/intel/whois?domain=`)"
- blueprint shows: getOneDomainWhoisRecord operation in filediff
- **adherent**: blueprint has dedicated operation for whois lookup.

### provider adherence

**provider setup**:
- vision says: `getDeclastructCloudflareProvider({ apiToken, accountId }, { log })`
- blueprint shows: provider/getDeclastructCloudflareProvider.ts in filediff
- **adherent**: provider factory matches vision signature.

**authentication**:
- vision says: apiToken (not apiKey - deprecated)
- blueprint shows: access/sdks/getCloudflareClient.ts in filediff
- **adherent**: has-pruned-backcompat review verified only apiToken is used.

### test adherence

- **adherent**: test count matches domain objects and operations count.

**integration tests**:
- criteria says: "against live cloudflare api"

- **adherent**: integration tests cover all api boundaries.

**acceptance tests**:
- criteria says: "declastruct plan with resources.ts"
- blueprint shows: declastruct.acceptance.test.ts + resources.acceptance.ts
- **adherent**: acceptance test covers plan/apply workflow.

### dao adherence
