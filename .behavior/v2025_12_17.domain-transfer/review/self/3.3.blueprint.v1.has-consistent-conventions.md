## detailed articulation: why each convention holds

### why domain object name (convention.1) holds

**extant evidence**:
```
DeclaredCloudflareDomainZone         → Zone
DeclaredCloudflareDomainDnsRecord    → DnsRecord
DeclaredCloudflareDomainRegistration → Registration
```

**pattern**: `DeclaredCloudflare` + `Domain` + `{Resource}`

**new name**: `DeclaredCloudflareDomainWhoisRecord` → `WhoisRecord`

**why it holds**: the new name follows the exact same pattern — prefix + Domain + resource name. no divergence.

### why directory name (convention.2) holds

**extant evidence**:
```
src/domain.operations/domainZone/
src/domain.operations/domainDnsRecord/
src/domain.operations/domainRegistration/
```

**pattern**: `domain{Resource}/`

**new name**: `domainWhoisRecord/`

**why it holds**: the new directory follows the exact same pattern — domain prefix + resource name. no divergence.

### why operation names (convention.3) hold

**extant evidence**:
```
getOneDomainZone, getAllDomainZones, setDomainZone, delDomainZone
getOneDomainDnsRecord, getAllDomainDnsRecords, setDomainDnsRecord, delDomainDnsRecord
getOneDomainRegistration, getAllDomainRegistrations, setDomainRegistration
```

**pattern**: `{verb}Domain{Resource}(s)`

**new name**: `getOneDomainWhoisRecord`

**why it holds**: the new operation follows the exact same pattern — verb + Domain + resource name. no getAll because WHOIS is single-domain lookup.

### why expandZoneRef divergence (convention.6) is correct

**extant name violates rules**: the forbidden term appears in the function name.

**why fix is correct**:
1. rule.forbid.term restrictions exist for good reason — overloaded terms cause confusion
2. "expand" precisely describes what the function does — expand ref to full object
3. fix aligns with ubiquitous language

**lesson**: when extant code violates rules, fix it — don't perpetuate violations.

### why unique key field order (convention.0.2) holds

**extant evidence for other domain objects**:
```
DeclaredCloudflareDomainZone: unique = ['name']
DeclaredCloudflareDomainRegistration: unique = ['name']
DeclaredCloudflareDomainDnsRecord (before): unique = ['zone', 'name', 'type']
```

**pattern**: fields ordered from most general (container) to most specific (discriminator)

**new key**: `['zone', 'name', 'type', 'content']`

zone (container) → name (record name) → type (A/CNAME/MX) → content (value)

**why it holds**: content is more specific than type — the same A record name can have multiple values. order follows general-to-specific.
