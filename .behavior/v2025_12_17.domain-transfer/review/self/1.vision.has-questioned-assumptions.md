## assumption 3: cloudflare API does NOT support domain registration/transfer

**what do we assume?** that cloudflare does not provide API endpoints for domain purchase or transfer initiation.

**evidence:** cloudflare API docs show registrar endpoints for:
- list domains in account (GET)
- get domain details (GET)
- update domain settings (PATCH)

no endpoints for: purchase, transfer initiation, auth code retrieval.

**what if opposite were true?** if API supported transfers, we wouldn't need HelpfulError guidance — we'd automate the full flow.

**verification needed:** re-confirm via API docs that no purchase/transfer endpoints exist.

**verdict: high confidence assumption, recommend verification**

## hidden assumption found: transfer steps are universal

the vision shows 10 steps for domain transfer. but these steps may vary by registrar.

**what if opposite were true?** some registrars might have different unlock flows, different auth code retrieval, different approval processes.

**risk:** the generic steps might not apply to all registrars.

**mitigation:** the vision already shows registrar name from WHOIS. the steps shown are cloudflare's generic guidance, not registrar-specific. user searches their registrar's docs.

**verdict: minor risk, already mitigated by design choice to not include registrar-specific links**
