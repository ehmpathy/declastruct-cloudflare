## potential divergence analysis

### question: why "Domain" in namespace?

all domain objects include "Domain" (e.g., `DeclaredCloudflareDomainZone`).

**analysis**:
- cloudflare has multiple services: dns, workers, pages, r2, etc
- "Domain" scopes to domain management (zones, dns, registrar)
- without "Domain": `DeclaredCloudflareZone` could be confused with other zone concepts
- with "Domain": `DeclaredCloudflareDomainZone` clearly indicates domain management

**verdict**: not a divergence. "Domain" is intentional namespace scoping.

---

### question: why no "Cloudflare" in operations?

operations use `getDomainZone`, not `getCloudflareZone`.

**analysis**:
- operations are invoked via cloudflare provider context
- cloudflare already implied by context
- shorter names improve readability
- domain objects include "Cloudflare" because they're exported publicly

**verdict**: not a divergence. operations scoped by context, objects scoped by name.

---

### question: why "DnsRecord" not "Record"?

domain object is `DeclaredCloudflareDomainDnsRecord`, not `DeclaredCloudflareDomainRecord`.

**analysis**:
- "Record" is too generic (could be database record, log record, etc)
- "DnsRecord" is unambiguous
- cloudflare API uses "dns_records" endpoint
- aligns with domain terminology

**verdict**: not a divergence. "DnsRecord" is more specific than "Record".

### operation naming — verified

from `src/domain.operations/domainZone/getAllDomainZones.ts`:
```ts
/**
 * .what = gets all domain zones from cloudflare account
 * .why = enables listing all zones for account management
 */
export const getAllDomainZones = async (
  context: ContextCloudflareApi,
): Promise<HasReadonly<typeof DeclaredCloudflareDomainZone>[]> => {
  const { client, accountId } = context.cloudflare;

  // fetch all zones using async iterator
  const zones: HasReadonly<typeof DeclaredCloudflareDomainZone>[] = [];
  for await (const zone of client.zones.list({ account: { id: accountId } })) {
    zones.push(castIntoDeclaredCloudflareDomainZone(zone));
  }

  return zones;
};
```

**why it holds**:
- name follows `{verb}{Namespace}{Resource}` = `getAllDomainZones`
- uses arrow function as required
- accepts context via (input?, context) pattern
- returns `HasReadonly<typeof DomainObject>[]` as declastruct pattern requires
- composes `castInto*` function (not inline transformation)
