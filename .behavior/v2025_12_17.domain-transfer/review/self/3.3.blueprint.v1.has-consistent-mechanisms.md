## detailed articulation: why each mechanism holds

### why cast function (mechanism.1) is not duplicate

**extant pattern verified**:
```
domainZone/castIntoDeclaredCloudflareDomainZone.ts
domainDnsRecord/castIntoDeclaredCloudflareDomainDnsRecord.ts
domainRegistration/castIntoDeclaredCloudflareDomainRegistration.ts
```

each cast function:
- takes cloudflare API response as input
- returns domain object instance
- extracts only relevant fields
- handles nullable fields with explicit null

**new cast function follows same pattern**:
```
domainWhoisRecord/castIntoDeclaredCloudflareDomainWhoisRecord.ts
```

- takes cloudflare WHOIS API response as input
- returns DeclaredCloudflareDomainWhoisRecord instance
- extracts domain, found, registrar, registrarName fields
- handles nullable registrar with explicit null

**why not duplicate**: each domain object needs its own cast — the API response shapes differ. no generic cast can serve all domain objects.

### why getOne operation (mechanism.2) is not duplicate

**extant pattern verified**:
```
domainZone/getOneDomainZone.ts → client.zones.get()
domainDnsRecord/getOneDomainDnsRecord.ts → client.dns.records.get()
domainRegistration/getOneDomainRegistration.ts → client.registrar.domains.get()
```

each getOne operation:
- accepts `{ by: { primary } | { unique } | { ref } }`
- calls specific cloudflare API endpoint
- returns domain object or null
- handles not-found gracefully

**new getOne follows same pattern**:
```
domainWhoisRecord/getOneDomainWhoisRecord.ts → client.intel.whois.get()
```

- accepts `{ domain: string }`
- calls cloudflare WHOIS API endpoint
- returns WhoisRecord or null

**why not duplicate**: each resource type has its own API endpoint. getOneDomainWhoisRecord cannot reuse getOneDomainZone — different APIs.

### why BadRequestError guidance (mechanism.3) is not duplicate

**extant usage verified**: helpful-errors is already a dependency. BadRequestError is used elsewhere in the codebase for user-actionable errors.

**new usage**: same error class, new message content (treestruct guidance for transfer/purchase).

**why not duplicate**: we reuse extant error class — the message content is the new part, which is required by criteria.

### why findsert diff check (mechanism.4) is not duplicate

**extant pattern verified**: searched all set* operations. none have diff check logic. this is a defect documented in has-questioned-5whys.

**new logic**: compare found.attributes with desired.attributes, throw BadRequestError if mismatch.

**why not duplicate**: this logic does not exist anywhere — it's a fix for a missing capability.

### why expandZoneRef (mechanism.5) is reuse not duplicate

**extant function verified**: `src/domain.operations/domainDnsRecord/` has an extant function that expands zone refs to zone IDs.

**blueprint action**: rename to expandZoneRef to comply with rule.forbid.term restrictions (extant name uses a forbidden term).

**why not duplicate**: this IS the extant function — we just rename it. no new mechanism created.
