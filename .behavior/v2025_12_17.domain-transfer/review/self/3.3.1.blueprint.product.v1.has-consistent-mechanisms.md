## potential duplication analysis

### question: could expandZoneRef be generic?

`expandZoneRef` resolves zone references to zone IDs for DNS record operations.

**analysis**:
- purpose: convert `Ref<Zone>` to zone ID for cloudflare API
- cloudflare-specific: requires zone lookup via cloudflare API
- no other ref types need expansion in this provider

**verdict**: not duplicate. cloudflare-specific, not a generic pattern.

---

### question: are cast functions redundant with `assure`?

`assure` from type-fns validates objects. cast functions transform AND validate.

**analysis**:
- cast functions: transform cloudflare API shape to domain object shape
- `assure` + `hasReadonly`: used within cast functions for validation
- not redundant: cast transforms, assure validates

**evidence from castIntoDeclaredCloudflareDomainZone.ts**:
```ts
return assure(
  DeclaredCloudflareDomainZone.as({ ... }),
  hasReadonly({ of: DeclaredCloudflareDomainZone }),
);
```

**verdict**: not duplicate. cast uses assure as composition, not replacement.

## code verification: actual file evidence

### castIntoDeclaredCloudflareDomainZone.ts

```ts
import { type HasReadonly, hasReadonly } from 'domain-objects';
import { UnexpectedCodePathError } from 'helpful-errors';
import { assure, isPresent } from 'type-fns';

export const castIntoDeclaredCloudflareDomainZone = (
  input: Zone,
): HasReadonly<typeof DeclaredCloudflareDomainZone> => {
  return assure(
    DeclaredCloudflareDomainZone.as({
      id: assure(input.id, isPresent),
      name: assure(input.name, isPresent),
      // ...
    }),
    hasReadonly({ of: DeclaredCloudflareDomainZone }),
  );
};
```

**verdict**: uses ecosystem patterns exactly as designed. no custom validation utilities.

### expandZoneRef.ts

```ts
import type { Ref } from 'domain-objects';
import { BadRequestError } from 'helpful-errors';
import { getOneDomainZone } from '@src/domain.operations/domainZone/getOneDomainZone';

export const expandZoneRef = async (
  zoneRef: Ref<typeof DeclaredCloudflareDomainZone>,
  context: ContextCloudflareApi,
): Promise<{ id: string; name: string }> => {
  if ('id' in zoneRef && zoneRef.id) {
    const zone = await getOneDomainZone({ by: { primary: { id: zoneRef.id } } }, context);
    if (!zone) BadRequestError.throw('zone not found for ref', { zoneRef });
    return { id: zone.id, name: zone.name };
  }
  // ...
};
```

**verdict**: reuses `getOneDomainZone` operation instead of duplicating zone lookup logic.

## why each mechanism holds

### why genDeclastructDao?

**alternative considered**: custom DAO classes

**why ecosystem wins**:
- `genDeclastructDao` handles type inference for byPrimary/byUnique
- provides consistent interface across all providers
- no boilerplate for findsert/upsert/delete mapping
- 3 DAOs, zero custom code

---

### why domain-objects DomainEntity/Literal?

**alternative considered**: plain interfaces

**why ecosystem wins**:
- `DomainEntity` provides `.as()` constructor with validation
- `static unique`/`static primary` enables declastruct to derive keys
- `static readonly` enables `hasReadonly` validation
- `static nested` enables hydration of nested objects
- plain interfaces lack all of these runtime capabilities

---

### why helpful-errors?

**alternative considered**: custom error classes

**why ecosystem wins**:
- `BadRequestError.throw()` one-liner pattern
- `.message` + `.metadata` structure for observability
- integrates with error handling ecosystem (log, monitor, etc)
- no custom error class definitions needed

---

### why type-fns assure/isPresent?

**alternative considered**: inline assertions

**why ecosystem wins**:
- `assure(value, isPresent)` returns typed value
- no `as` casts needed
- composable with `hasReadonly`
- throws `UnexpectedCodePathError` on failure

---

### why expandZoneRef not generic?

**question**: could this be a generic `expandRef` utility?

**analysis**:
- `expandZoneRef` resolves zone refs for dns record operations
- cloudflare API requires zone_id, but users declare by zone name
- this is cloudflare-specific glue, not a reusable pattern

**why not generic**:
- only one ref type needs expansion (zone)
- expansion requires cloudflare API lookup (not pure)
- no other declastruct providers have this pattern
- making it generic adds complexity for single use case

**verdict**: cloudflare-specific. not a duplicate of anything.

## conclusion

**no duplicate mechanisms found**.

the blueprint reuses ecosystem patterns:
- `genDeclastructDao` from declastruct for DAO generation
- `DomainEntity`/`DomainLiteral` from domain-objects for domain modeling
- `RefByUnique`/`refByUnique` from domain-objects for references
- `hasReadonly`/`assure` from domain-objects/type-fns for validation
- `BadRequestError`/`UnexpectedCodePathError` from helpful-errors for errors

no custom utilities folder exists. all mechanisms delegate to ecosystem packages.

### why this matters

each ecosystem pattern provides:
- **type safety**: generic types infer correctly
- **consistency**: same patterns across all providers
- **testability**: ecosystem tools have their own test coverage
- **maintenance**: ecosystem updates benefit all consumers

custom mechanisms would:
- duplicate ecosystem functionality
- require separate test coverage
- drift from ecosystem conventions
- add maintenance burden

**verdict**: blueprint is consistent with codebase patterns.
