| `process.exit` | hard exits that abort test runner | 0 matches |
| `jest.fn()` + no assertions | mocks that short-circuit behavior | reviewed, none found |
| hardcoded `return` before assertions | silent pass | reviewed, none found |

### credential pattern verification

read each integration test file to verify the credential check pattern:

```typescript
// domainZone.integration.test.ts:17-20
const hasCredentials =
  process.env.CLOUDFLARE_API_TOKEN && process.env.CLOUDFLARE_ACCOUNT_ID;
const describeWithCredentials = hasCredentials ? describe : describe.skip;

describeWithCredentials('with live cloudflare api', () => {
  // tests run here when credentials present
});
```

**why this pattern is safe**:
1. credentials check is explicit and at top of file — visible to anyone who reads the test
2. `hasCredentials` is a boolean that evaluates to true when both env vars are set
3. tests inside `describeWithCredentials` run when credentials present — no silent bypass
4. credential absence is logged via jest output ("5 skipped") — visible in test results
5. no `if (hasCredentials) return` pattern that would silently pass without assertions

**why it holds**: credential bypasses would silently pass tests that should fail. but the pattern here uses `describe.skip` — which shows up in test output. jest reports "5 skipped" when credentials are absent. this is explicit, not silent.
